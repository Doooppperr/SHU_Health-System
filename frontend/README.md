# Health System Frontend

Vue 3 + Vite 6 前端，以“公开门户 + 三角色工作台”为页面结构。普通用户、机构管理员和系统管理员拥有独立首页、菜单、路由和接口调用，不共享业务后台。

## 环境与安装

要求 Node.js 20+、npm 10+。

~~~powershell
npm ci
~~~

## 页面与路由

### 公开页面

| 路由 | 页面 |
|---|---|
| / | 康迹 HealthHub 公开门户：项目介绍、核心功能、使用流程、隐私提示、关于我们 |
| /login | 独立品牌化登录页 |
| /register | 普通用户注册或填写邀请码的机构工作人员注册 |
| /403 | 无权访问 |
| 其他未知路径 | 404 页面 |

### 普通用户工作台

登录后入口为 /dashboard，菜单包括：

- 健康总览
- 健康档案与档案详情
- OCR 报告录入
- 指标趋势
- 亲友授权
- 体检机构与套餐
- 我的评论
- 个人中心

档案表单中的机构和套餐均可“暂不选取”。选择来源机构时会提示：档案确认后，标准化指标会只读开放给对应机构管理员。

档案列表和详情页统一显示 `health+数字` 编号，内部路由和 API 仍使用数字主键。档案可直接手工录入，也可从列表/详情进入 OCR 页面，为同一档案上传新报告；待确认结果可刷新后继续，也可放弃而不影响原档案。

### 机构管理员工作台

登录后入口为 /org/dashboard，菜单包括：

- 机构运营总览
- 机构资料维护
- 机构相册管理
- 体检套餐管理
- 本机构健康档案
- 本机构指标趋势

机构健康页面只展示后端返回的脱敏只读数据，不提供档案修改、补录、删除和原始报告入口。

### 系统管理员工作台

登录后入口为 /admin/dashboard，菜单包括：

- 系统运营总览
- 机构、套餐与相册管理
- 邀请码管理
- 用户与角色管理
- 全局档案监管
- 评论审核

系统管理员可签发机构邀请码、查看邀请码状态、撤销机构管理员，以及停用或恢复机构和套餐。

## 角色路由保护

路由元数据明确限定 user、institution_admin、admin。登录、刷新令牌或重新获取当前用户后，前端会按后端返回的真实角色跳转：

- user → /dashboard
- institution_admin → /org/dashboard
- admin → /admin/dashboard

跨角色访问会进入 /403；未登录访问工作台会跳转到 /login；已登录用户访问登录或注册页会返回所属工作台。后端仍执行独立权限校验，前端守卫不是唯一安全边界。

## 机构相册

机构管理员和系统管理员页面均支持：

- JPEG、PNG、WebP 上传。
- 每张最多 5 MB、每机构最多 8 张。
- 拖拽排序。
- 第一张自动作为封面。
- 删除图片。

实际格式、大小、EXIF 清理和数量上限由后端再次验证。

## AI 智能助手

- 公开门户、登录和注册页显示访客 AI，仅回答公开系统导览。
- 普通用户工作台显示完整健康 AI；打开侧栏时不请求、不展示档案列表。
- 普通问题直接发送。用户点击“引用档案”，或后端对“我的报告、历史趋势”等问题返回 `select_records` 动作时，才在当前消息附近按需加载选择器。
- 档案列表支持同一所有者的不限数量多选；点击“智能分析（N）”后打开侧栏。单档覆盖全部指标，多档展示服务端计算的指标变化和趋势。
- 机构管理员和系统管理员工作台不显示健康 AI。
- 选择档案后需确认指标会发送到配置的 DeepSeek API；选择与同意只对当前请求生效，请求完成、取消或切换选择后立即重置。
- 用户消息先乐观显示，AI 通过 SSE 拼接正文；发送中防重复提交，并提供取消、失败保留和可重试状态。
- 对话仅存入当前标签页 sessionStorage；结束对话或退出登录时清理。界面最多保留 40 条消息，发送历史最多 20 条，敏感档案上下文不会自动进入无关的后续历史。
- 悬浮球位置与侧栏宽度保存在 localStorage，不包含对话或健康数据。
- DeepSeek API Key 只由后端读取，前端代码和浏览器存储不包含密钥。
- 桌面端侧栏展开时主页面按比例缩放，最低 0.7；空间不足或窄屏改为遮罩对话框，并把键盘焦点移入面板，不再挤压左侧导航。

接口事件、错误码和 OCR 状态机见 [`../项目文档/AI与OCR开发说明.md`](../项目文档/AI与OCR开发说明.md)。

主要文件：

- src/components/AiAssistant.vue：访客/用户 AI 交互界面。
- src/stores/aiChat.js：会话、按需选档、逐请求同意、流式发送、取消和重试状态。
- src/utils/aiStageLayout.js：侧栏宽度、主页面缩放和遮罩模式计算。
- src/utils/recordDisplayId.js：`health+数字` 展示编号格式化。
- src/utils/roles.js：角色与工作台映射。
- src/layouts/WorkspaceLayout.vue：三套菜单与通用工作台外壳。
- src/api/admin.js、src/api/org.js、src/api/dashboards.js：角色专属接口。

## 开发启动

~~~powershell
npm run dev
~~~

默认地址：http://127.0.0.1:5173

Vite 将 /api 和 /uploads 代理到 Flask 后端 http://127.0.0.1:5050。项目根目录也提供：

~~~powershell
.\scripts\start-frontend-dev.ps1
~~~

## 构建与预览

~~~powershell
npm run build
npm run preview
~~~

本机生产预览默认地址：http://127.0.0.1:4173

项目根目录也提供 scripts/start-frontend-prod.ps1。
