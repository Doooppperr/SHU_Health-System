## AI/OCR 优化与用户反馈闭环（2026-07-13）

本轮在三角色结构之上完成并发布以下更新：

- AI 改为 SSE 流式协议，事件为 `meta/status/delta/action/done/error`；用户消息先显示，可取消、失败保留和重试。
- 移除打开侧栏时的常驻选档。普通问题不读取档案，手动“引用档案”或 `select_records` 动作才按需加载，并对每次请求重新选择和同意。
- 档案列表支持同一所有者不限数量多选并发起“智能分析（N）”。单档覆盖全部指标，多档趋势由服务端确定性计算后交给模型解释。
- 历史达到 20 条消息时使用本地确定性裁剪/摘要合并，不再进行额外模型摘要调用；连接、读取和总截止默认 5/30/60 秒。
- 修复 Waitress 因 SSE 响应携带 `Connection` hop-by-hop 头而返回 500 的问题。
- 现有档案可直接补传 OCR。新报告先作为带随机 `attachment_id` 的待确认附件，刷新可恢复、可放弃，确认成功后才替换正式报告并 upsert 指标。
- OCR 上传、确认、取消、重传和档案删除引入快照 CAS，修复旧页面覆盖、切档响应污染和并发文件孤儿风险；同时修复英文 `reference` 范围进入指标值导致无效映射的问题。
- 档案展示编号统一为 `health+数字`，内部数据库主键和 API 参数仍为整数，不变更 schema。
- AI 面板改为桌面主页面缩放；最低比例 0.7，空间不足切换为 overlay，并补齐动态焦点迁移。
- 本地 OCR 样图、数据库、上传目录、视频和 PDF 保持不进入 Git。

### OCR `region-v2` 通用解析加固（2026-07-13）

在上述已发布版本之后，根据三种明显不同版式的 OCR 联调反馈继续完成通用解析加固：

- 华为通用表格响应改为按每个 `table` 区域独立解析，解决多个区域行列索引从 0 重置后互相覆盖的问题；表头扫描不再限制前四行。
- 表格提取与非表格文本兜底并行，不再因表格已经产出三个以上字段就忽略无边框列表或英文行式结果；同时支持无表头代码/名称/数值表的列推断。
- 指标映射改为保守策略：英文短代码必须精确命中代码或别名，冲突、低置信度和非法数值统一标记 `requires_review`。字典外项目保持未匹配，不猜成相似指标。
- 数值规范化拒绝有歧义的逗号写法，避免小数点误丢；接受 `μ/µ/u` 微摩尔字形和常见 `mol·L⁻¹` 等价记法，但不进行未配置单位换算。
- 既有档案确认界面默认 `replace_ocr`，只清理新报告没有覆盖的旧 OCR 来源指标并保留手工指标；用户可切换 `merge`，服务端缺省仍为 `merge` 以兼容旧客户端。
- 确认页展示解析器版本，候选值可编辑；需复核候选默认忽略，报告元数据/未配置项目单独展示，减少“元数据未映射等于指标识别失败”的误解。

本次本地验证基线为后端 143 项、前端 18 个文件 102 项、Vite production build、`pip check` 和生产依赖审计通过。三份不含真实健康信息的合成报告经真实华为云 OCR 和同一 `region-v2` 链路分别映射 10/9/8 个已配置指标，冲突与需复核均为 0。这个 smoke 只验证已配置字典项目跨多种布局的表现，不承诺字典外任意项目自动映射。

### 关怀模式统一缩放（2026-07-13）

用户联调发现旧关怀模式通过分别增大字体 token、控件高度和侧栏宽度实现，公开门户导航因此出现逐字换行和横向挤压。现已改为画布级统一放大：桌面端最高 1.12 倍，文字、控件和间距保持原比例；可用设计宽度不足 1180 时自动降低倍率或回退到 1 倍。关怀倍率与 AI 侧栏原有缩小倍率在 `aiStageLayout` 中合并，保证主页面可视宽度不变，并避免两套变换相乘后侵入侧栏。

新增 4 项布局计算测试，覆盖独立关怀放大、与 AI 侧栏组合、窄屏回退和 overlay 不缩放底层页面。更新后前端基线为 18 个文件、106 项测试通过，Vite production build 通过；真实浏览器回归覆盖 1920 桌面、AI 同开和 1024 窄屏，导航均无横向溢出。

### 普通用户机构封面展示修复（2026-07-13）

机构相册上传、排序和接口序列化原本均正常，`/api/institutions` 与详情接口已经返回第一张相册图对应的 `cover_image_url`；问题来自普通用户的机构列表与详情模板没有消费任何图片字段。现新增复用封面组件并接入两处页面，字段优先级为 `cover_image_url → logo_url → images[0].image_url`，兼容当前接口、迁移后的旧 logo 与最小图片集合响应。图片使用惰性加载、明确替代文本和固定比例裁切；无图片或加载失败时显示可访问占位，不影响资料、套餐和评论操作。

## 已确认并落地的产品决策

- 访客首先进入 `/` 公开门户，再由独立登录、注册页面进入系统。
- 普通用户、机构管理员、系统管理员分别进入 `/dashboard`、`/org/dashboard`、`/admin/dashboard`。
- 两类管理员不进入普通用户的个人档案、亲友授权或带档案上下文的健康 AI 页面。
- 档案的机构和套餐均可不选；选择套餐时可推导机构，清空机构时同步清空套餐。
- 只有来源于某机构且状态为 `confirmed` 的档案才向该机构管理员提供结构化只读投影。
- 每个机构最多一个机构管理员和一个当前邀请码；邀请码单次使用、永不过期且数据库只保存哈希。
- 机构和套餐使用软停用；机构相册最多 8 张，服务端解码、重编码并清除 EXIF。
- 健康报告由鉴权接口读取，公共上传路径只开放机构图片。
- SQLite 继续作为唯一数据库，不引入 MySQL、PostgreSQL 或远程托管。

## 主要实现记录

### 身份、权限与工作台

- `users.role` 支持 `user / institution_admin / admin`。
- `users.managed_institution_id` 为唯一可空外键，并通过检查约束与角色保持一致。
- 服务端逐请求读取当前用户角色和机构绑定；角色撤销后无需等待旧 JWT 过期即可失权。
- 前端为三角色提供独立菜单、路由守卫、首页和 API 模块，并提供 403、404 页面。
- 普通档案、趋势、亲友和用户评论接口只允许普通用户；机构健康数据使用独立只读接口；全局档案监管使用 admin 专属接口。

### 机构、邀请码与相册

- 系统管理员可新增、编辑、停用和恢复机构，并管理机构套餐、相册、邀请码和机构管理员。
- 机构管理员只可维护其绑定机构的资料、套餐和相册。
- 停用机构会撤销当前机构管理员和未使用邀请码，同时保留历史档案、套餐、评论与图片。
- 邀请码签发时只返回一次明文；重发覆盖当前行并立即使旧哈希无效，消费与用户绑定在事务内完成。
- 图片上传限制为 JPEG、PNG 或 WebP、每张 5 MB、每机构 8 张；排序 0 的图片自动成为封面。
- 普通用户机构列表和详情消费 `cover_image_url` 展示封面，并为旧 logo、缺图和加载失败提供兼容回退。

### 健康档案、指标与文件

- 手工录入和 OCR 确认均支持“暂不选取”机构与套餐。
- 新关联只能选择启用中的机构和套餐；已停用来源仍可作为历史关系保留，且不阻止修改档案其他字段。
- 指标名称继续映射标准字典，数值会清理单位文本、箭头、空格和全角符号；不支持的单位差异进入人工确认，不静默换算。
- 机构管理员只能查看本机构已确认档案的脱敏结构化数据、异常项和趋势，不能获取联系方式、上传人、原始报告，也没有写接口。
- 机构健康数据访问只写不含指标值的结构化运行日志，不新增数据库审计表。
- 原始报告使用 `/api/records/{id}/file` 鉴权返回；`/uploads` 仅允许 `institution_images` 登记的机构图片，报告和孤儿文件均拒绝。
- 普通用户接口不允许修改既有档案归属人，避免亲友代理在授权期内永久转移档案。
- 报告和相册删除会补偿清理本地文件；公共上传路由还会核对相册数据库白名单，孤儿文件不可公开读取。
- `display_id=health{id}` 为响应层派生值，不新增数据库列；删除档案不会重排其他现存档案。
- `ocr_raw_text` 可以暂存 `_pending_attachment`；待确认报告不是正式业务事实，确认后指标仍规范化保存到 `health_indicators`。

### AI 与公开门户

- 公开门户包含项目介绍、核心能力、使用流程、隐私提示和关于我们，并提供固定登录/注册入口。
- 未登录访客可使用公开系统导览；只有普通用户可以向 AI 提交其有权限的已确认档案 ID。
- 机构管理员和系统管理员界面不挂载健康 AI，后端也拒绝其提交档案上下文。
- AI 延续健康科普、人工客服和急症分流边界，不诊断、不推荐处方药，也不建立聊天业务表。
- 普通问题不加载档案；档案引用、同意和敏感历史只对当前明确请求有效。
