# 康康健健 HealthDoc：项目需求与技术方案

> 文档状态：schema v6、体检预约履约、套餐变更审核、机构报告永久归档、亲友只读授权、AI/OCR 与本地双通道 RAG 当前完整需求与技术方案，更新于 2026-07-16。

## 1. 项目目标

构建一个可在个人电脑直接运行、同时支持公开服务、个人健康管理和体检机构协作的 B/S 平台。系统使用 Vue 3、Flask 与本地 SQLite，围绕四类长期目标展开：

1. 用户掌握自己的身份资料、日常健康测量和体检结果；
2. 机构对正式体检报告的生产、复核和准确提交负责；
3. 亲友访问建立在明确授权和最小字段披露之上；
4. 管理员治理平台对象，但不接触健康内容；AI/OCR 辅助工作，不替代医疗判断和人工复核。

## 2. 页面体系与角色

系统采用“公开门户 + 三工作台”。

| 角色 | 默认入口 | 当前核心权限 |
|---|---|---|
| 访客 | `/` | 项目介绍、登录注册和访客 AI 导览 |
| 普通用户 `user` | `/dashboard` | 个人资料、自测、体检预约、时间线、趋势、报告、亲友只读查看、机构、评论和健康 AI |
| 机构账号 `institution_admin` | `/org/dashboard` | 所属机构资料、相册、预约履约、套餐审核申请及体检报告生产 |
| 系统管理员 `admin` | `/admin/dashboard` | 机构、套餐变更审批、相册、邀请码、账号状态和评论审核 |

三类登录角色具有独立导航、路由和后端接口。两类管理员不进入普通用户健康中心；系统管理员不拥有健康内容查询权限。

## 3. 功能需求

### 3.1 公开门户与认证

- `/` 展示项目介绍、核心能力、使用流程、隐私提示和关于我们。
- 顶部提供登录、注册；已登录用户显示与角色匹配的“进入工作台”。
- 登录和注册保留图片验证码。
- 普通用户可直接注册；机构工作人员必须提交当前有效邀请码。
- 服务端生成普通用户唯一健康身份码，不接收客户端自定义值。
- 登录成功按角色跳转；停用账号不能登录、刷新或继续调用受保护接口。
- 访问令牌、刷新令牌和当前用户按标签页保存在 `sessionStorage`；同一浏览器的多个标签页可分别登录不同账号，登录、刷新和退出互不覆盖。

### 3.2 普通用户健康中心

- 总览显示当前健康内容摘要，不直接暴露亲友隐私。
- 个人资料包含用户名、联系方式、真实姓名、生日、性别、过敏史和既往史；健康身份码只读。
- 日常测量允许身高、体重、心率、体温、血氧、空腹血糖，同日可多次录入。
- 普通用户在独立“体检预约”页选择机构、已审核启用套餐和今天起 30 天内日期；所选日期展示剩余名额，满额时禁止提交。
- 同一用户同一天全平台只能保留一条有效预约；未履约预约可由用户取消并立即释放名额。
- 机构报告从预约生成并提交后自动进入普通用户健康时间线、报告列表和趋势。
- 时间线合并预约全生命周期、自测和已发布报告；预约创建后立即显示，并使用同一预约事件随状态同步更新；报告列表和详情只读。
- 趋势支持本人及授权亲友，按每日有效值展示数据来源。
- 用户继续浏览机构、套餐、相册与评论，并管理自己的评论。

### 3.3 预约、履约与自动归档

- 健康身份码仍由系统生成并只读；预约直接绑定当前注册用户，机构不再手工输入或搜索健康身份。
- 预约创建状态为 `unfulfilled`（未履约），创建后立即按预约日期进入本人及获授权亲友健康时间线。机构可随时置为不可恢复的 `invalidated`（已失效），同一预约事件随状态同步更新。
- 机构确认用户到检后状态变为 `awaiting_report`（待上传报告），此后用户不能取消，机构也不能再标记失效。
- 只有待上传报告预约可建立草稿；用户、机构、日期和套餐取预约快照并禁止修改。报告正式提交后，报告发布与预约转 `fulfilled` 在同一事务完成。
- 机构每日上限为空表示不限量，只允许设置正整数；降低上限不取消既有预约，超额日期只停止新增。容量校验和预约写入必须串行化，避免超卖。

### 3.4 机构报告生产

- 同一机构可有多个账号，每个账号只绑定一个机构；机构 ID 由服务端从账号绑定取得。
- 机构账号可维护机构资料和相册，设置每日预约上限，并提交套餐变更申请。
- 报告只可从待上传报告预约手工或 OCR 新建草稿。
- 草稿内只可修改指标；身份、日期和套餐保持预约快照，指标来源记录为 `manual` 或 `ocr`。
- OCR 的低置信度、冲突、无效数值、单位问题和未映射字段必须人工复核。
- 锁定前再次校验预约绑定用户仍启用；锁定后报告正文和指标永久不可编辑，草稿原文件立即删除。
- 提交时再次验证账号仍启用，成功直接进入 `published`。
- 提交前必须明确确认；提交后的报告永久归档、只读开放，不可修改、撤下或删除。
- 机构账号不能查看其他机构报告，也不能通过请求参数切换机构。

### 3.5 系统管理员后台

- 总览只显示平台账号、角色、机构、邀请码和评论等非健康统计。
- 管理机构和相册，执行机构软停用/恢复；套餐只读展示。
- 审批机构提交的套餐新增、修改、下架和恢复申请。通过后自动生效，驳回不修改当前套餐；管理员不能直接改套餐。
- 为机构签发或重新生成邀请码；明文仅在签发响应出现一次，数据库只保存哈希。
- 管理账号启用状态；删除普通用户需明确确认，删除机构账号保留报告快照。
- 审核评论公开状态。
- 不提供报告、自测、指标、时间线、趋势、个人健康资料或 AI 健康上下文接口。

### 3.6 邀请码与机构账号生命周期

- 每机构维护一个当前邀请码行；状态为 `active/used/superseded`。
- 重新生成直接覆盖当前行并清除旧使用元数据，旧明文立即失效。
- 邀请码消费、账号创建和机构绑定在同一事务内完成。
- 同机构可重复签发并注册多个工作人员账号，但每个邀请码只能成功消费一次。
- 删除机构账号不删除机构或历史报告，报告保留创建用户名快照。

### 3.7 机构相册与机构生命周期

- 每机构最多 8 张 JPEG、PNG 或 WebP 图片，单张不超过 5 MB。
- 支持拖拽排序，第一张自动作为封面；无图或加载失败显示占位。
- 服务端实际解码、修正方向、清除 EXIF，并校验存储路径。
- 机构使用软停用；套餐上下架也必须经过审核，保留历史报告和预约快照。
- 公开 `/uploads` 只返回数据库已登记机构图片，不开放草稿报告文件或整个上传目录。

### 3.8 亲友、评论与隐私

- 普通用户继续建立亲友关系，由被添加方控制授权状态。
- 授权只提供时间线、已发布报告和趋势的只读投影；时间线和趋势页面均提供本人/已授权亲友成员切换。
- 亲友不能代传体检报告、代记日常测量或修改对方任何健康数据。
- 亲友响应排除真实姓名、生日、性别、联系方式、健康身份码、过敏史、既往史、内部匹配 ID 和受检者姓名快照。
- 撤权或账号停用后，下一次请求立即重新鉴权。
- 用户拥有某机构已发布匹配报告后才可评论；系统管理员审核公开状态。

### 3.9 AI

- 访客 AI 只做公开导览；机构账号和管理员不使用健康上下文。
- 普通问题不查询或发送健康数据。
- 需要个人报告时，通过 `select_records` 或用户主动引用按需加载报告列表。
- 每次引用都重新选择并明确同意；选择和同意不延续到无关消息。
- 可分析报告必须属于本人或已授权亲友且仍为 `published`；同一请求必须属于同一人。
- 上下文排除个人资料和健康身份码，只包含必要报告指标与服务端每日有效趋势事实。
- 服务端计算排序、首次/最新值、极值、异常、缺失、同日来源优先和变化，模型只解释事实。
- SSE 事件为 `meta/status/delta/action/done/error`，支持取消、失败保留和重试。
- AI 只做科普、一般建议、产品客服和急症分流，不诊断、不推荐处方药。
- 聊天与分析不写数据库；浏览器当前标签页可暂存有限界面消息。
- 私人档案不向量化；公共知识使用批准清单、FastEmbed 中文嵌入和 Qdrant Local。访客及管理员只检索公开系统说明，医学语料只供登录普通用户。
- 应用启动和用户请求期间不联网抓取语料；远端哈希变化必须隔离并继续使用旧批准索引，显式批准后才能原子切换 collection alias。
- 首版 UI 不展示来源详情，后端只返回是否使用 RAG、检索状态和来源数量；审计日志不得记录问题、片段或健康值。

## 4. 技术方案

| 层级 | 选型 |
|---|---|
| 前端 | Vue 3、Vite 6、Vue Router、Pinia、Element Plus、Axios、ECharts 6 |
| 后端 | Flask 3、Flask-SQLAlchemy、Flask-JWT-Extended、Flask-Cors、bcrypt |
| 本地数据库 | SQLite schema v6，`PRAGMA user_version` 管理结构版本 |
| 文件 | 机构图片长期保存；OCR 原文件只在报告草稿期临时保存 |
| OCR | 本地 Mock 或华为云通用表格 OCR，`region-v2` 解析与保守字典映射 |
| AI/RAG | DeepSeek V4 Flash、SSE、FastEmbed、Qdrant Local、本地 FAQ、安全规则与确定性健康事实 |
| 测试 | Pytest、Vitest、Vue Test Utils、jsdom |

ECharts 使用 `echarts/core` 按需注册并通过趋势路由懒加载。健康时间线与趋势由服务端生成权限专用 DTO，前端不复用管理员或机构完整对象。

## 5. 数据与权限原则

- 角色、账号启用状态和机构绑定每次请求查库验证。
- 普通用户健康身份与机构绑定互斥，由数据库检查约束保证。
- 机构报告与日常自测独立建模，不把用户自测伪装成机构报告。
- 报告一旦锁定不可变，提交后永久保留并持续进入用户健康视图。
- 趋势计算先把不同数据库驱动返回的 `DATE` 值统一为自然日，再合并机构报告与日常自测。
- 精确匹配不执行模糊查询，不返回候选用户。
- 亲友、用户、机构和管理员使用不同 DTO，遵循最小字段原则。
- AI、邀请码、图片和临时报告文件均由服务端执行最小权限校验。
- 文件写入失败或事务失败时补偿清理；锁定成功后删除临时报告文件。
- production 默认监听本机地址；对外监听必须使用强 JWT、强管理员密码、防火墙和 HTTPS。

## 6. 数据库与迁移要求

- 本地数据库目标为 SQLite schema v6。
- 新空库直接创建 v6；非 v6 数据库拒绝直接启动，v5 使用升级脚本全量保留迁移。
- 本地升级保留系统管理员主键与密码哈希，并在安全重建前生成完整备份。
- 临时库必须通过结构、完整性和外键校验后才能原子替换正式库。
- 生产 openGauss 数据同步必须先备份，再由当前 SQLite→openGauss 迁移脚本创建完整 schema、复制全部表、重置序列并核对逐表行数；不能仅依赖 `create_all()` 修改旧库。

## 7. 非功能需求

- 支持 Windows PowerShell，本地不依赖额外数据库服务。
- 密码、邀请码只保存哈希；密钥来自被 Git 忽略的环境变量。
- 健康身份码、报告指标和个人资料不进入公开日志。
- 三工作台支持桌面和移动端，具备 403、404、加载、错误和空状态。
- AI 侧栏桌面最低页面缩放 0.7，空间不足切换焦点可达的 overlay。
- 关怀模式最高 1.12 倍统一缩放，并与 AI 侧栏共同计算可用画布。
- AI 默认连接/读取/总超时 5/30/60 秒，失败不能无限加载。
- schema、模型、接口、前端、测试和文档必须同步更新。

## 8. 当前实现与验收结论

- 公开门户、独立认证、三角色工作台、机构服务、亲友、评论、AI 和关怀模式均已实现。
- schema v6 的个人资料、自测、预约履约、套餐审核、机构报告永久归档和亲友只读边界已实现。
- 机构多账号、邀请码覆盖生命周期、账号停用/恢复/删除和报告创建者快照已实现。
- 亲友只读 DTO、机构同日优先和 AI 新数据边界已实现。
- OCR 已迁移到机构草稿，同时保留多布局解析、保守映射和人工复核；锁定删除临时原文件。
- 公共知识 RAG 的批准清单、原子同步、quarantine、检索安全降级和前端检索状态已实现，私人健康数据保持在业务数据库通道。
- 当前合成 SQLite 快照 `user_version=6`、`integrity_check=ok`、外键违规 0，包含 5 个普通用户的丰富时间线、25 条五状态预约、4 条套餐审核历史和完整三角色账号矩阵。
- 后端 31 项、前端 15 个文件 88 项、生产构建、依赖审计、44 条 RAG 黄金查询和数据库全量迁移冒烟通过。
- 2.0 计划文件按要求保留，后续由用户自行决定删除。
