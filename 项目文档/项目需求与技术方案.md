# 体检评价与健康档案系统：项目需求与技术方案

## 1. 项目目标

构建一个可以在个人电脑上直接运行的 B/S 健康管理平台。用户通过浏览器管理个人和亲友体检档案，所有业务数据保存在项目本地 SQLite 数据库中。

## 2. 角色

- 普通用户：管理个人资料、体检档案、亲友关系、授权和评论。
- 管理员：管理用户、审核评论，并拥有普通用户能力。

## 3. 功能需求

### 3.1 用户认证

- 注册、登录和退出。
- 图片验证码。
- JWT Access Token 与 Refresh Token。
- 用户查看和修改自己的资料。

### 3.2 机构与套餐

- 浏览体检机构列表和详情。
- 查看机构提供的体检套餐。
- 用户上传过该机构档案后才允许评论。

### 3.3 健康档案

- 创建、查看和删除档案。
- 区分档案归属人和上传人。
- 记录体检机构、套餐、体检日期和报告文件。
- 手动添加、修改和删除健康指标。
- 根据参考范围自动判断数值是否异常。

### 3.4 OCR

- 支持图片和 PDF。
- 支持本地 Mock 与真实 OCR API 两种模式。
- 解析结果先进入候选确认页。
- 用户确认映射后才正式写入健康指标。

### 3.5 亲友授权

- 添加、改名和删除亲友关系。
- 被添加方控制授权状态。
- 授权后允许对方代传档案和查看趋势。

### 3.6 趋势分析

- 按档案归属人和指标查询时间序列。
- 展示折线图、参考区间、最新值、最小值和最大值。

### 3.7 评论和后台管理

- 用户提交评分与评论。
- 用户查看和删除自己的评论。
- 管理员控制评论是否公开。
- 管理员查看、修改和删除用户。

## 4. 技术方案

| 层级 | 选型 |
|---|---|
| 浏览器端 | Vue 3 + Vite + Element Plus |
| 状态与路由 | Pinia + Vue Router |
| 网络请求 | Axios |
| 图表 | ECharts |
| 服务端 | Flask |
| ORM | Flask-SQLAlchemy |
| 数据库 | SQLite |
| 鉴权 | Flask-JWT-Extended |
| 本机演示服务 | Waitress |
| 自动化测试 | Pytest |

## 5. 本地运行架构

```text
浏览器
  │
  ▼
Vue/Vite :5173（演示模式为 :4173）
  │ /api 与 /uploads 代理
  ▼
Flask/Waitress :5050
  ├─ backend/instance/health_system.db
  └─ backend/uploads/
```

系统没有远程数据库依赖。开发模式和本机演示模式共享本地 SQLite 文件。

## 6. 数据设计原则

- 关系表达到 3NF；基于已识别函数依赖的核心表满足 BCNF。
- 使用外键表达实体关系。
- 使用唯一约束保证候选键和防止重复数据。
- 使用检查约束限制角色、状态、评分和数值范围。
- 跨表权限和业务规则由服务端事务校验。
- 测试使用内存 SQLite，与实际本地数据隔离。

## 7. 非功能需求

- Python 3.10+、Node.js 18+。
- 支持 PowerShell 一键启动。
- 数据库和上传文件可以直接复制备份。
- 密码使用 bcrypt 哈希。
- 敏感 OCR 凭据只保存在被 Git 忽略的 `backend/.env`。
