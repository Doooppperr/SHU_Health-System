# 康康健健 HealthDoc 数据库规范化说明

> 文档状态：SQLite schema v4 当前规范化与完整性说明，更新于 2026-07-16。

## 1. 规范化目标

账号、机构、套餐、邀请码、图片、指标字典、自测、机构报告、报告指标、亲友和评论分别建模。当前关系以 3NF 为最低目标，并在已识别函数依赖范围内满足 BCNF。

- `self_measurements` 保存用户本人记录的时间点测量。
- `institution_reports` 与 `report_indicators` 保存机构锁定并自动归档的正式体检事实。
- schema v4 没有用户体检登记实体，也不保存等待匹配数据。

## 2. 范式检查

### 1NF

- 每条自测和报告指标单独成行，不使用动态指标列或日内数组。
- 套餐、邀请码、图片、亲友和评论分别建表。
- `indicator_dicts.aliases` 与 `institution_reports.ocr_diagnostics` 是不参与候选键和连接的有意 JSON 文档值。

### 2NF

- 报告指标的值、异常和来源依赖完整候选键 `(report_id, indicator_dict_id)`。
- 套餐属性依赖套餐实体，图片位置依赖 `(institution_id, sort_order)`。
- 亲友授权依赖完整用户对 `(user_id, friend_user_id)`。

### 3NF

- 自测与报告指标只引用字典 ID，单位、范围和临床意义留在指标字典。
- 账号引用机构，不复制机构属性；报告引用套餐，不复制套餐价格和描述。
- 报告创建者用户名快照是独立历史事实，用于账号删除后的审计显示。
- `report{id}` 是主键派生展示值，不重复落库。

## 3. 候选键与 BCNF

| 表 | 候选键 |
|---|---|
| `users` | `username`、非空 `email`、非空 `health_id` |
| `institutions` | `(name, branch_name)` |
| `packages` | `(institution_id, name)` |
| `institution_invites` | `institution_id`、`code_hash` |
| `institution_images` | `(institution_id, sort_order)`、`storage_key` |
| `indicator_categories` | `name` |
| `indicator_dicts` | `code`、`(category_id, name)` |
| `friend_relations` | `(user_id, friend_user_id)` |
| `institution_reports` | 未撤下状态的 `(institution_id, subject_health_id, exam_date)` 部分候选键 |
| `report_indicators` | `(report_id, indicator_dict_id)` |

各非平凡函数依赖的决定因素均为超键。`users.managed_institution_id` 不是候选键，因为一个机构允许多个工作人员账号。

## 4. 有意保留的快照

- `created_by_username_snapshot` 保存报告创建时的用户名，不代表当前账号属性。
- `subject_name_snapshot` 与 `subject_health_id` 保存机构提交的身份事实；成功归档后仍保留用于审计，但用户和亲友 DTO 隐藏敏感匹配字段。
- `ocr_diagnostics` 保存解析候选与复核诊断；正式健康指标仍在 `report_indicators`。

这些字段语义独立，不是可从当前外键对象无损重算的缓存，因此不构成更新异常。

## 5. 完整性规则

### 数据库约束

- 角色限定为 `user/institution_admin/admin`，角色与机构绑定、健康身份互斥。
- 用户名、非空邮箱、健康身份码唯一。
- 自测值与套餐价格非负，评论评分为 1–5。
- 机构图片排序位置唯一且限制为 0–7。
- 报告状态仅为 `draft/locked/published/withdrawn`。
- 同机构、健康身份码和体检日期最多一个未撤下报告。
- 一个报告的每个标准指标最多一行。

### 服务端事务

- 自测仅允许白名单指标且只能操作本人数据。
- 邀请码消费与机构账号创建原子提交。
- 报告只有草稿可写；锁定成功后删除临时文件。
- 锁定前按健康身份码和姓名验证唯一启用注册普通用户；提交再次校验，成功立即发布。
- 亲友授权只影响只读查询，不产生任何代传、代记或修改能力。
- 普通用户删除需显式确认；机构账号删除保留报告与创建者快照。

## 6. 权限边界

- 本人资料、亲友只读 DTO、机构报告 DTO 和管理员账号 DTO 分开序列化。
- 亲友投影排除真实姓名、联系方式、健康身份码、病史和内部归属字段。
- 机构账号只能操作数据库绑定机构的报告，不可搜索或浏览用户健康档案。
- 管理员接口不读取报告指标、自测、时间线或个人资料。
- AI 只读取本人或已授权亲友的 `published` 报告与每日有效趋势事实。

## 7. 趋势不落表

每日有效趋势由查询实时确定：同日 `published` 机构指标优先，否则使用该日最后一次自测；报告撤下后自然回退。如果持久化最终值，报告发布、撤下及自测 CRUD 都需维护缓存，反而引入更新异常。

## 8. 生命周期

- 机构和套餐软停用；机构账号物理删除后报告创建者外键置空、快照保留。
- 普通用户删除级联其自测、归属报告和报告指标。
- 报告撤下保留行与指标，但用户正文、趋势和 AI 统一过滤。
- 身份不匹配的锁定报告由机构继续核对或撤下，不进入等待队列。
- 图片和草稿文件按数据库引用清理，孤儿扫描限制在上传目录。

## 9. 物理结果

当前正式本地数据库是从空库生成的 schema v4 合成快照：`user_version=4`、`integrity_check=ok`、外键违规 0。5 个普通用户均有丰富自测和至少 2 份已发布报告，覆盖同日多测、机构指标优先、撤下回退、异常标记和亲友只读查看。

当前模型满足 3NF，并在已识别函数依赖范围内满足 BCNF；快照与 OCR JSON 均具有独立业务语义和明确权限边界。
