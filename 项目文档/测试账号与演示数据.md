- 本文件与合成演示数据库可以保存在本地并提交到 GitHub `main`，方便拉取后直接验收。

- 不要把本文件中的密码用于真实账号、真实数据库或其他系统。
- `backend/instance/health_system.db` 是可提交的合成演示快照；同目录时间戳备份仍受 `.gitignore` 排除。

## 7. 推荐验收账号顺序

1. 使用 `test1` 在“体检预约”查看所选日期余量、五种状态和取消操作，再查看健康时间线与趋势。
2. 使用 `test2` 查看多份永久归档报告持续进入报告列表、时间线和趋势。
3. 使用 `test5` 查看多指标和异常标记展示。
4. 使用 `institution1_staff1` 验证确认到检、标记失效、每日上限和“信息审核”。
5. 使用 `institution2_staff1` 从“待上传报告”预约验证 OCR 草稿、锁定和履约归档。
6. 使用 `demo_admin` 在“审核记录”通过或驳回套餐申请，并确认管理员不能直接修改套餐。

## 8. 重新生成说明

演示数据由 `backend/app/seed.py` 在全新空库首次启动时生成。需要重建时，应先停止后端、备份现有 SQLite，再删除或移走 `backend/instance/health_system.db` 并启动应用。不要在包含真实数据的数据库上使用合成快照覆盖操作。
