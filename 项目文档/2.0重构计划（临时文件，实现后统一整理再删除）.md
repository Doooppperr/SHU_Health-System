# 康康健健 HealthDoc 2.0 重构计划与实现结果

> 临时文件，更新于 2026-07-16。按要求继续保留，最终由用户自行删除。

## 1. 当前目标

HealthDoc 2.0 围绕个人长期健康视图、机构报告生产、亲友只读授权和三角色权限隔离构建。当前业务结论：

- 普通用户维护个人资料并记录本人日常测量。
- 体检机构为已经注册的普通用户创建、复核和提交标准化体检报告。
- 机构提交报告后，系统按健康身份码和真实姓名即时自动归档到对应用户。
- 普通用户无需登记体检，不存在等待匹配、补登记或保留期限。
- 亲友授权只允许查看对方的时间线、报告和趋势，不能代传、代记或修改。
- 系统管理员治理账号、机构、邀请码和评论，不读取健康内容。

## 2. 角色与权限

### 2.1 普通用户

- 注册、登录、刷新和退出。
- 查看只读健康身份码，维护本人资料。
- 新增、修改和删除本人六类日常测量。
- 查看本人健康总览、健康时间线、体检报告和指标趋势。
- 查看机构资料、套餐、相册及公开评论。
- 建立亲友关系，申请查看亲友健康数据。
- 控制其他亲友对本人健康视图的只读授权。
- 对已拥有报告的机构发表评论。
- 在逐请求同意后使用本人或已授权亲友的报告进行健康 AI 分析。

普通用户不能创建、上传、编辑或删除机构报告，也不能操作亲友的日常测量。

### 2.2 机构账号

- 一个机构可绑定多个同权限账号。
- 维护本机构资料、套餐和相册。
- 手工或通过 OCR 创建本机构报告草稿。
- 复核受检者身份、标准化指标和异常标志。
- 锁定报告、提交并永久自动归档。
- 查看本机构各报告状态数量。

机构账号不能切换到其他机构，不能浏览普通用户列表、日常测量、亲友关系、其他机构报告或完整健康档案。

### 2.3 系统管理员

- 管理机构、套餐、相册和机构启用状态。
- 签发机构工作人员邀请码。
- 管理账号启用状态并执行受控删除。
- 审核公开评论。
- 查看非健康平台统计。

系统管理员没有报告、指标、自测、健康时间线、趋势、个人健康资料或健康 AI 入口。

## 3. 普通用户健康视图

### 3.1 健康总览

- 显示时间线事件、日常测量和已归档机构报告数量。
- 提供“记录日常测量”快捷入口。
- 展示最近健康事件。

### 3.2 日常测量

当前自测白名单：

- 身高 `HEIGHT`
- 体重 `WEIGHT`
- 心率 `HR`
- 体温 `TEMP`
- 血氧 `SPO2`
- 空腹血糖 `FBG`

同日允许多次测量，保留实际测量时间和全部原始值。所有写操作固定归属当前登录用户，不接受他人用户 ID。

### 3.3 健康时间线

时间线由服务端统一生成，包含：

- 本人日常测量；
- 机构已归档报告；

亲友时间线使用脱敏只读 DTO，不返回真实姓名、健康身份码、联系方式、病史、内部归属 ID 或受检者姓名快照。

### 3.4 指标趋势

- 趋势按日期输出一个每日有效值。
- 同日存在 `published` 机构指标时优先采用机构值。
- SQLite 与 openGauss 返回的日期值先统一为自然日后再排序和合并。
- 机构报告缺少目标指标时采用当日最后一次自测。
- 支持本人和当前已授权亲友。

## 4. 机构报告自动归档

### 4.1 草稿

机构填写：

- 受检者真实姓名；
- 健康身份码；
- 体检日期；
- 可选套餐；
- 一个或多个标准化指标。

草稿可手工创建或由 OCR 创建。OCR 低置信度、冲突、未知字段、非法数值和单位问题必须人工复核。

### 4.2 锁定

- 报告至少包含一项指标。
- 受检者姓名和健康身份码必须对应唯一启用注册普通用户；失败时草稿保持可编辑。
- 锁定后正文和指标不可再编辑。
- OCR 临时原文件在事务提交后物理删除。
- 清除不需要长期保留的原始解析内容。

### 4.3 提交与即时归档

机构锁定前执行身份校验，提交 `locked` 报告时再次执行同一查询：

```text
health_id = subject_health_id
real_name = subject_name_snapshot
role = user
is_active = true
```

匹配成功后在同一事务中：

- 写入 `matched_user_id`；
- 写入 `submitted_at` 和 `published_at`；
- 状态改为 `published`；
- 报告立即进入用户报告列表、时间线、趋势和可选 AI 上下文。

锁定前身份不存在、姓名不一致、账号停用或角色不正确时：

- 返回 409；
- 报告保持可编辑 `draft`；
- 不返回候选用户；
- 不创建等待数据。

锁定后若账号被停用，提交返回 409 且报告保持 `locked`。

### 4.4 永久归档

- 提交前机构必须明确确认。
- `published` 报告永久归档，不可修改、撤下、删除或回到草稿。
- 报告持续进入对应用户和已授权亲友的报告列表、时间线、趋势与 AI 数据范围。

## 5. 亲友只读授权

关系模型包含请求方、被查看方、关系名称和授权状态。

- 请求方添加亲友并等待被查看方授权。
- 被查看方可随时开启或关闭只读授权。
- 授权后请求方可以在时间线和趋势页面切换到对方，并查看脱敏时间线、已发布报告和趋势。
- 关闭授权、删除关系或账号停用后，下一次请求立即失效。
- API 不返回旧的可管理/代传集合。
- 前端不显示“可代传”等文案。
- 自测写接口只操作当前登录用户；机构报告写接口只属于机构账号。

## 6. schema v5

### 6.1 健康核心表

- `self_measurements`
- `institution_reports`
- `report_indicators`
- `indicator_categories`
- `indicator_dicts`

当前结构不存在 `exam_registrations`。

### 6.2 报告状态

```text
draft -> locked -> published
```

允许状态只有 `draft/locked/published`。

### 6.3 报告关键字段

- `institution_id`
- `created_by_user_id`
- `created_by_username_snapshot`
- `subject_name_snapshot`
- `subject_health_id`
- `exam_date`
- `package_id`
- `matched_user_id`
- `status`
- `ocr_diagnostics`
- `temporary_file_url`
- `locked_at/submitted_at/published_at`

同机构、健康身份码和体检日期只允许一份报告。

## 7. 当前 API

### 普通用户

- `GET/POST /api/self-measurements`
- `PUT/DELETE /api/self-measurements/<id>`
- `GET /api/health/timeline`
- `GET /api/health/trends/<indicator_id>`
- `GET /api/exam-reports`
- `GET /api/exam-reports/<id>`
- `GET/POST/PUT/DELETE /api/friends...`

没有 `/api/exam-registrations`。

### 机构账号

- `GET /api/org/dashboard`
- `GET/POST/PUT /api/org/reports...`
- `POST /api/org/reports/ocr`
- `POST /api/org/reports/<id>/lock`
- `POST /api/org/reports/<id>/submit`
- 机构资料、相册和套餐接口

### 管理员

- 机构、套餐、相册和邀请码管理
- 账号查询、启停和删除
- 评论审核

## 8. 前端结果

普通用户侧栏包括：

- 健康总览
- 健康时间线
- 日常测量
- 指标趋势
- 亲友授权
- 体检机构
- 我的评论
- 个人资料

普通用户没有“登记体检”按钮、侧栏、路由或页面。机构详情只展示资料、套餐、相册和评论。

机构报告页状态筛选为草稿、已锁定、已归档；锁定报告按钮显示“提交并归档”，提交前明确提示不可撤回。

## 9. 隐私和安全

- 角色、账号状态和机构绑定逐请求查库。
- 普通用户健康身份码由服务端生成并只读。
- 机构不能搜索候选用户。
- 亲友只得到脱敏只读健康视图。
- 系统管理员不访问健康内容。
- OCR 原文件不公开并在锁定时删除。
- AI 私人上下文只来自已授权 `published` 报告和每日有效趋势。
- 私人数据不进入 Qdrant；向量库只保存批准的公共知识。
