# 康康健健 HealthDoc 服务器部署与同步

> 当前发布工具支持 schema v7 平台代码发布，以及显式选择的合成演示数据库全量同步。数据库同步会先创建离线物理备份，失败时恢复原数据库和上一代码版本。

## 1. 服务器拓扑

- 公网入口：Apache 监听 80，前端构建文件位于 `/var/www/html/`。
- API：Apache 将 `/api` 与 `/uploads` 反向代理到 `127.0.0.1:5050`。
- 后端：`healthdoc.service` 以非登录系统用户 `healthdoc` 运行，代码入口为 `/opt/healthdoc/current/backend`。
- 邮件：`healthdoc-notifications.service` 以同一受限用户常驻运行，每 5 秒条件领取 Outbox；发送前把事件载荷转换为连续中文正文，不暴露 JSON 内部字段；服务与 API 共同重启、失败自动拉起。
- 数据库：openGauss 容器 `healthdoc-gaussdb`，仅绑定 `127.0.0.1:5432`，数据持久化到 `/var/lib/healthdoc/opengauss`。
- 上传文件：`/var/lib/healthdoc/uploads`。
- RAG 运行目录：`/var/lib/healthdoc/rag`，由 `healthdoc` 用户持有，保存 Qdrant Local、嵌入模型和缓存。
- 生产环境变量：`/etc/healthdoc/healthdoc.env`，权限为 root-only，不进入 Git。

本地开发使用 `backend/instance/health_system.db`。服务器通过 `DATABASE_URL` 使用 openGauss。普通代码发布不上传 SQLite；只有显式指定 `-SyncDemoDatabase` 时才生成一致性临时快照和受限演示附件包并全量同步，完成后立即从服务器删除临时 SQLite 与附件包。

## 2. 两种发布模式

本地发布入口要求 PowerShell 7 或更高版本，避免 Windows PowerShell 5.1 改写传给 Python 的多行快照校验参数。

普通代码发布：

```powershell
.\scripts\deploy-server.ps1
```

它保留服务器现有数据库和上传目录，更新时间戳代码版本与前端构建，并从仓库批准清单同步公共 RAG 知识；RAG 同步失败时保留原环境和当前版本。

合成演示数据库、附件与真实邮件配置全量同步：

```powershell
.\scripts\deploy-server.ps1 -SyncDemoDatabase -SyncMailSettings
```

该模式明确清空 openGauss 应用表并导入 Git 中的合成快照，只适用于用户已确认可覆盖的数据环境。脚本会停止 API 和通知 worker、停止数据库容器并备份 `/var/lib/healthdoc/opengauss`，同时备份上传目录和 root-only 环境文件；导入脚本逐表核对行数与报告归属，并只接受固定目录中的 9 个水印 PNG。数据库、附件、RAG、API 或通知 worker 任一环节失败都会恢复数据库、上传目录、环境和上一代码版本。

`-SyncMailSettings` 从被 Git 忽略的本地 `backend/.env` 读取 SMTP 白名单字段，经 SSH 临时文件写入 `/etc/healthdoc/healthdoc.env` 后立即删除。它要求 SMTP 主机、端口、用户名、授权码和发件人完整，且 `NOTIFICATION_EMAIL_DRY_RUN=0`；授权码不会出现在 Git、发布命令或文档中。

`upgrade_local_database.py` 只用于 SQLite 旧库升级；跨数据库全量同步使用 `migrate_sqlite_to_gaussdb.py --replace`。

`reset_v7_demo_data.py` 只用于本地演示库，不属于服务器发布或生产迁移流程，任何生产环境都不得执行。

## 3. schema v7 生产迁移验收项

### 数据结构

- `users` 包含健康身份、个人资料和 `is_active` 字段，并满足角色/机构/健康身份约束；
- 同一机构允许多个机构账号；
- 存在健康领域、指标领域关联、套餐版本、套餐版本领域关联；
- 存在同行安排、参与者预约、预约事件、日期容量、候补订阅、空位通知与 outbox；
- 存在自主测量、机构归档、数值指标、文本结果和报告附件；
- 指标字典包含自主测量开放标志，健康时间线支持体检/自主记录投影；
- 邀请码状态满足 `active/used/superseded`；
- 活动报告的唯一性规则正确；
- 报告创建账号删除时外键置空、快照保留；
- 普通用户删除时健康数据按设计级联。
- 应用启动通过 SQLAlchemy Inspector 校验 openGauss 表、列和命名约束，不执行 SQLite 专属的 `sqlite_master` 查询。

### 数据与文件

- 系统管理员账号和密码哈希正确；
- 机构、套餐版本、领域关联、相册与评论数据处置符合迁移方案；
- 上传目录只保留数据库引用的机构图片、报告附件和必要临时文件；
- 公共 `/uploads/reports/...` 返回 404；
- 报告锁定后临时原文件删除；

### 权限与业务

- 普通用户、机构账号、管理员分别进入正确工作台；
- 管理员不能访问健康内容；
- 机构账号只能访问绑定机构；
- 同行安排的参与者校验、容量扣减、日期冲突和取消释放正常；满额候补、空位事件与 outbox 可追踪且不重复；
- 报告归档时再次校验预约用户启用状态，并与预约履约同步完成；数值指标、文本结论和附件均按权限访问；
- 套餐申请、管理员审批及新版本生效正常，历史预约继续读取原版本快照；
- 提交后的报告保持永久归档，机构端没有撤下接口，用户时间线、趋势和 AI 持续读取；
- 亲友 DTO 不包含个人资料、健康身份或内部匹配字段；
- 亲友没有代记自主测量、编辑报告或修改他人数据的接口；
- AI 和报告识别服务密钥只来自服务器环境文件。

## 4. 发布流程

在项目根目录执行普通代码发布：

```powershell
.\scripts\deploy-server.ps1
```

发布脚本会执行后端依赖检查/测试、前端依赖审计/测试和生产构建，打包不含 `.env`、SQLite、上传文件、`项目文档` 和虚拟环境的发布包，通过 SSH 上传，创建时间戳版本，显式同步批准的公共 RAG 语料，并在 API 与通知 worker 都健康后完成切换。远程来源短暂超时时，只能使用 SHA-256 与清单完全一致的既有批准缓存；首次同步没有合格缓存时仍会失败回滚。因此《测试账号与演示数据》保留在本地和 GitHub，不进入服务器运行目录；显式演示同步会另行上传经校验的临时 SQLite 和水印附件，导入后删除临时文件。

仅在已单独完成全部测试且保留验证记录时才可使用：

```powershell
.\scripts\deploy-server.ps1 -SkipTests
```

普通代码发布不执行数据迁移。只有带 `-SyncDemoDatabase` 的显式命令会在同一受控发布单元中备份并覆盖演示数据库与附件；`-SyncMailSettings` 才会改变服务器 SMTP 配置。

## 5. 发布前检查

本地执行：

```powershell
Push-Location .\backend
.\.venv\Scripts\python.exe -m pytest -q
.\.venv\Scripts\python.exe -m pip check
.\.venv\Scripts\python.exe -m pip_audit -r requirements.txt
.\.venv\Scripts\python.exe .\scripts\evaluate_rag.py
Pop-Location

Push-Location .\frontend
npm test
npm run build
npm audit --omit=dev
Pop-Location

git diff --check
```

另需确认：

- 生产发布分支和提交已评审；
- `.env`、SQLite、数据库备份、上传文件、开发文档、真实报告和密钥未进入代码发布包；
- 合成 SQLite 只在全量同步模式临时上传，且远端导入完成后删除；
- 全量迁移脚本已通过临时目标库逐表行数、完整性、外键和双向引用冒烟；
- RAG 批准清单、黄金查询和服务器持久目录权限已验证；
- 回滚版本、数据库恢复点和上传目录快照可用；
- Apache、systemd 与容器磁盘空间充足；
- 运维人员已知停机窗口和验证清单。

## 6. 查看状态与日志

```bash
sudo systemctl status healthdoc healthdoc-notifications apache2 docker
sudo journalctl -u healthdoc -n 100 --no-pager
sudo journalctl -u healthdoc-notifications -n 100 --no-pager
sudo docker ps --filter name=healthdoc-gaussdb
sudo docker logs --tail 100 healthdoc-gaussdb
curl -fsS http://127.0.0.1:5050/api/health
curl -fsS http://127.0.0.1/api/health
```

健康检查成功只表示进程可用，不代表迁移和权限验收完成。发布后仍需执行三角色、健康总览、两类时间线、同行预约、候补、套餐版本、机构归档、文件和 AI/报告识别 smoke。

## 7. 从本地安全连接数据库

数据库端口不直接开放公网。使用桌面数据库工具时先建立 SSH 隧道：

```powershell
ssh -L 15432:127.0.0.1:5432 ubuntu@111.229.87.94
```

随后连接 `127.0.0.1:15432`。数据库名和用户名按服务器 root-only 配置读取，密码不得写入仓库、命令历史或文档。

## 8. 备份范围

至少备份：

- openGauss 逻辑备份；
- `/var/lib/healthdoc/opengauss` 的受控基础设施备份；
- `/var/lib/healthdoc/uploads`；
- `/etc/healthdoc/healthdoc.env`；
- 当前 `/opt/healthdoc/current` 指向的发布版本和上一个可回滚版本。

数据库运行期间优先使用 openGauss `gs_dump` 做一致性逻辑备份，不直接复制正在写入的数据目录。环境文件和数据库备份应加密、限制权限并设置保留周期。

恢复演练必须在独立目录或测试实例进行，至少验证：

- 账号可登录且密码哈希有效；
- 外键、唯一约束和行数符合预期；
- 机构图片可读取；
- 报告附件与临时文件引用一致；
- 同行预约、候补通知、套餐版本、报告归档和趋势规则正常。

## 9. 回滚原则

- 应用代码与数据库 schema 必须作为同一个发布单元考虑。
- 如果数据库迁移已经提交，不能只把代码 symlink 切回不兼容版本。
- 回滚前停止写入，恢复数据库备份、上传目录和兼容代码版本。
- 任何失败都先保存日志和现场快照，再执行预演过的回滚步骤。
- 不在生产环境临时手工修改表来绕过迁移脚本。

## 10. HTTPS 与网络安全

健康资料属于敏感信息。服务器转为正式对外收集真实数据前必须：

- 绑定域名并配置 Apache 443；
- 使用 Certbot 或可信证书；
- 将 80 重定向到 HTTPS；
- 仅开放必要端口；
- openGauss 继续只绑定回环地址；
- 删除或重置演示账号，禁止使用演示管理员密码和本机演示标志；
- 配置强 JWT 密钥和强管理员密码；
- 对日志、备份和环境文件执行最小权限控制。

## 11. 当前结论

schema v7 平台代码、合成演示快照、水印附件、通知 worker、全量迁移脚本和回滚发布路径组成同一个可验证发布单元。服务器若承载真实数据，只允许普通代码发布或另行设计增量迁移；不得执行本地演示数据重置，也不得使用 `-SyncDemoDatabase` 覆盖真实业务库。
