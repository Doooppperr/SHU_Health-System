# 康康健健 HealthDoc 本地运行与演示指南

本指南适用于当前“公开门户 + 三角色工作台 + 个人健康资料/自测 + 机构报告永久归档 + 亲友只读授权/评论 + AI/OCR + SQLite schema v5”完整版本。

## 1. 环境要求

- Windows 10/11、PowerShell 5.1+
- Python 3.10+
- Node.js 20+
- npm 10+
- 本地运行无需安装 MySQL、PostgreSQL 或其他数据库服务

## 2. 首次准备

在项目根目录执行：

```powershell
Set-Location .\backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

Set-Location ..\frontend
npm ci

Set-Location ..
if (-not (Test-Path .\backend\.env)) {
    Copy-Item .\backend\.env.example .\backend\.env
}
```

默认数据库：

```text
backend/instance/health_system.db
```

仓库跟踪这一份只含合成数据的 SQLite 演示快照，拉取后可直接验收。时间戳数据库备份、`backend/.env`、`backend/uploads/` 和运行时 RAG 目录不受 Git 跟踪。

## 3. 环境变量

离线演示建议保持：

```env
OCR_USE_MOCK=1
AI_USE_MOCK=1
```

真实 AI 需填写：

```env
AI_USE_MOCK=0
DEEPSEEK_API_KEY=你的密钥
```

真实华为云 OCR 需填写：

```env
OCR_USE_MOCK=0
HUAWEI_OCR_ENDPOINT=
HUAWEI_OCR_AK=
HUAWEI_OCR_SK=
HUAWEI_PROJECT_ID=
```

真实密钥只能写入 `backend/.env`。不要写入 `.env.example`、源码、截图、终端记录或文档。

## 4. 数据库检查与升级

启动前检查：

```powershell
Set-Location .\backend
if (Test-Path .\instance\health_system.db) {
    .\.venv\Scripts\python.exe .\scripts\upgrade_local_database.py --check-only
} else {
    Write-Host "数据库尚未创建，首次启动会创建 schema v5。"
}
Set-Location ..
```

预期当前输出包括：

```text
user_version=5
expected_user_version=5
schema_current=yes
```

如果版本不是 3，停止后端后执行：

```powershell
Set-Location .\backend
.\.venv\Scripts\python.exe .\scripts\upgrade_local_database.py
Set-Location ..
```

脚本会先生成 `before-schema-v5` 备份，在临时库完成结构、行数、完整性和外键验证后原子替换。

```text
backend/instance/health_system.before-schema-v5-*.db
```

不要在确认不再需要前删除该备份。

## 5. 开发模式

在项目根目录执行：

```powershell
.\scripts\start-full-dev.ps1
```

脚本会打开后端 PowerShell 窗口，并在当前窗口启动前端：

- 前端：<http://127.0.0.1:5173>
- 后端：<http://127.0.0.1:5050>
- 健康检查：<http://127.0.0.1:5050/api/health>

Vite 会把 `/api` 和 `/uploads` 代理到后端 `5050`。

## 6. 本机生产演示

```powershell
.\scripts\start-full-prod.ps1
```

- 前端：<http://127.0.0.1:4173>
- 后端：<http://127.0.0.1:5050>

脚本默认只监听回环地址。首次运行若缺少 JWT 密钥，会在被 Git 忽略的 `backend/.env` 生成随机值，并设置仅当前进程有效的本机演示标志。

不得把本机演示配置直接暴露到局域网或公网。对外监听至少需要：

- `JWT_SECRET_KEY` 长度不少于 32；
- `DEFAULT_ADMIN_PASSWORD` 长度不少于 12；
- 防火墙访问限制；
- HTTPS；
- 完成生产数据库 schema v5 迁移演练。

## 9. 推荐完整演示顺序

### 9.1 公开门户与认证

1. 打开 `/`，查看项目介绍、核心功能、使用流程、隐私提示和关于我们。
2. 打开登录、注册页，确认图片验证码正常加载。
3. 普通用户注册不需要邀请码；机构工作人员注册必须填写管理员签发的当前邀请码。
4. 用不同角色登录，确认分别进入用户、机构和管理员工作台。
5. 尝试跨角色 URL，确认进入 403 或被后端拒绝。
6. 在同一浏览器打开两个标签页，分别登录不同账号并刷新页面，确认两个标签页仍各自保持原账号；在一个标签页退出，另一个标签页不受影响。

### 9.2 普通用户完整流程

用 `test1` 登录：

1. 在“个人资料”确认健康身份码只读，可修改真实姓名、生日、性别、联系方式、过敏史和既往史。
2. 在“健康总览”查看当前摘要。
3. 在“日常测量”新增体重、心率、体温、血氧或空腹血糖；同日新增两次体重。
4. 在“健康时间线”查看自测与机构自动归档报告按时间合并。
5. 在“指标趋势”查看体重：体检日应使用机构报告值 `71.9`，而不是同日两次自测；其他日期取当天最后自测。
6. 查看“体检报告”列表和详情，确认报告及指标完全只读。
7. 浏览“体检机构”，确认列表和详情显示相册封面、资料与套餐；无图时显示占位。
8. 在“亲友授权”查看或维护关系；授权后可切换亲友时间线/趋势，但不能代传、代记或修改对方数据。
9. 在“我的评论”查看或删除本人评论；评论资格要求已有该机构发布的报告。
10. 打开健康 AI，先问普通问题，确认不加载报告；再主动引用报告并明确同意，观察 SSE 状态和回答。

### 9.3 机构账号完整流程

用 `institution1_staff1` 登录：

1. 查看机构运营总览及各报告状态数量。
2. 修改所属机构资料。
3. 新增、修改或停用本机构套餐。
4. 上传机构图片、拖拽排序并确认第一张成为普通用户封面。
5. 在“体检报告”手工创建草稿，填写受检者姓名、健康身份码、体检日期和可选套餐。
6. 新增标准指标并确认参考范围与异常标志。
7. 锁定报告；身份不一致时锁定被拒绝且草稿仍可修改，校验通过后锁定内容不可编辑。
8. 提交报告：再次校验账号仍启用后立即自动归档。
9. 提交前确认页面明确提示不可撤回；提交后确认机构端不再显示撤下操作，普通用户的报告、时间线、趋势和 AI 持续使用该报告。
10. 使用同机构另一账号登录，确认它能管理同一机构数据；其他机构账号不能访问该报告。

### 9.4 OCR 报告流程

在机构报告页选择 OCR 上传：

1. 保持 `OCR_USE_MOCK=1` 完成离线演示。
2. 提交 PDF/PNG/JPEG/WebP 及受检者信息。
3. 查看 `parser_version`、候选映射、置信度、冲突、未匹配和需复核项目。
4. 对需复核项手工修正或补充，不把未知字段直接当成标准指标。
5. 确认草稿至少有一项指标后锁定。
6. 锁定后确认报告不可修改，临时原文件已删除，用户侧无文件入口。
7. 提交后验证报告立即进入对应用户时间线；身份不一致会在锁定前被拦截。

### 9.5 系统管理员完整流程

用本机管理员账号登录：

1. 总览只显示账号、角色、机构、邀请码和评论等平台统计。
2. 新增、编辑、停用和恢复机构；维护套餐与相册。
3. 为机构生成邀请码，立即复制页面仅显示一次的明文。
4. 重新生成邀请码，确认旧码失效。
5. 查看账号，停用后确认该账号不能登录或刷新；再恢复。
6. 删除机构账号，确认历史报告仍保留创建用户名快照。
7. 删除普通用户时确认必须显式二次确认，并级联清理其健康数据。
8. 审核评论公开状态。
9. 确认管理员页面没有报告、自测、时间线、趋势、个人健康资料或健康 AI。

## 10. 自动归档演示建议

1. 记录普通用户只读健康身份码和真实姓名。
2. 机构账号创建报告，填写完全一致的姓名和健康身份码，录入指标并锁定。
3. 提交后应返回 `matched`，报告立即进入该用户报告列表、时间线和趋势。
4. 另建草稿并故意修改姓名或健康身份码；锁定应返回 409，草稿仍可修正，不会出现等待状态或候选用户。

## 11. AI 模式

Mock 模式不消耗外部额度，适合自动化与课堂演示。真实模式演示前：

1. 在 `backend/.env` 设置 `AI_USE_MOCK=0` 和 `DEEPSEEK_API_KEY`；
2. 重启后端；
3. 先测试公开 FAQ，再测试登录用户普通问答；
4. 最后选择已发布报告并确认同意；
5. 不在截图、录屏或日志中展示密钥、健康身份码和真实健康数据。

AI 只提供科普和一般建议。出现急症关键词时应优先展示就医/急救提示。

### 11.1 本地公共知识 RAG

RAG 只保存批准的公共知识，不保存用户问题、聊天、OCR 原文、用户/报告 ID 或健康指标值。首次在 `backend` 目录显式执行：

```powershell
.\.venv\Scripts\python.exe .\scripts\rag_sync.py sync
.\.venv\Scripts\python.exe .\scripts\evaluate_rag.py
```

同步成功后在 `backend/.env` 设置 `RAG_ENABLED=1` 并重启后端。模型、Qdrant Local、缓存、来源快照与 quarantine 位于 `backend/instance/rag/`，不进入 Git。应用启动和用户请求不会联网更新语料；批准来源变化时先进入 quarantine，审核后再显式批准和同步。

## 14. 备份与恢复

停止后端后至少备份：

- `backend/instance/health_system.db`
- `backend/instance/health_system.before-schema-v5-*.db`
- `backend/instance/health_system.before-demo-refresh-*.db`
- `backend/uploads/`
- `backend/.env`（单独加密保存）

恢复步骤：

1. 停止后端和所有可能写库的脚本；
2. 先复制当前数据库和上传目录作为恢复前快照；
3. 把目标备份复制为 `backend/instance/health_system.db`；
4. 执行 `upgrade_local_database.py --check-only`；
5. 启动后端并检查 `/api/health`；
6. 验证账号、机构图片、报告状态和外键。

不要直接覆盖正在写入的 SQLite 文件。

## 15. 常见问题

### 后端提示 schema upgrade required

停止后端，先执行 `--check-only`，再运行升级脚本。确认备份文件生成后重新启动。

### 前端请求失败或代理到错误端口

确认后端监听 `127.0.0.1:5050`，访问 `/api/health`。使用完整启动脚本时不要手工改代理；自定义端口需同步 `VITE_API_PROXY_TARGET`。

### 登录或刷新返回账号不可用

账号可能被管理员停用。使用管理员工作台恢复后重新登录，旧 token 不应绕过状态检查。

### 邀请码无法使用

确认机构启用、邀请码为当前 `active`、复制时没有多余空格，并且该码没有被使用或重新生成覆盖。

### 报告无法锁定，提示找不到用户

逐字核对真实姓名和健康身份码，并确认目标账号是启用的普通用户。系统不提供模糊候选；失败时草稿保持可编辑。

### 报告无法编辑

只有 `draft` 可编辑。锁定后内容不可变；提交归档后不可撤下、删除或回到草稿。

### 普通用户看不到报告

确认报告是 `published`，`matched_user_id` 指向当前用户。用户只读接口不返回草稿或锁定报告。

### 趋势值与同日最后自测不同

同日存在已发布机构指标时，机构值优先；该指标未出现在报告中时使用当天最后自测。

### OCR 返回错误或候选很少

确认文件格式、20 MiB 总大小、PDF 页数和 OCR 配置。未知或低置信度项目不会自动映射，需要人工复核或扩充指标字典。

### AI 提示未配置

离线演示设 `AI_USE_MOCK=1`。真实模式需填写 `DEEPSEEK_API_KEY` 并重启后端。

### Waitress 提示 JWT_SECRET_KEY

推荐从根目录运行生产演示脚本，它会为回环演示生成密钥。直接调用 Waitress 时必须手工配置至少 32 字符的密钥。

### 可以直接运行服务器发布脚本吗

可以使用根目录 `scripts/deploy-server.ps1`；同步演示数据库时显式增加 `-SyncDemoDatabase`。脚本会先备份 openGauss 与当前版本，失败时回滚。
