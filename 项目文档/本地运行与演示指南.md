# 本地运行与演示指南

## 1. 首次准备

```powershell
Set-Location .\backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

Set-Location ..\frontend
npm install

Set-Location ..
```

本地数据库已经位于：

```text
backend/instance/health_system.db
```

不需要安装独立数据库服务。

## 2. 开发模式

在项目根目录执行：

```powershell
.\scripts\start-full-dev.ps1
```

访问 `http://127.0.0.1:5173`。

该脚本会：

1. 在新 PowerShell 窗口启动 Flask 后端。
2. 在当前窗口启动 Vite 前端。
3. 将前端 `/api` 和 `/uploads` 请求代理到 `127.0.0.1:5050`。

## 3. 本机演示模式

```powershell
.\scripts\start-full-prod.ps1
```

访问 `http://127.0.0.1:4173`。

后端由 Waitress 启动，前端使用 Vite Preview，数据库仍是本地 SQLite。

## 4. 单独启动

```powershell
.\scripts\start-backend-dev.ps1
.\scripts\start-frontend-dev.ps1
```

或：

```powershell
.\scripts\start-backend-prod.ps1
.\scripts\start-frontend-prod.ps1
```

## 5. 默认账号

```text
admin / admin123
```

## 6. 建议演示顺序

1. 登录并查看机构与套餐。
2. 创建一份手动档案并添加指标。
3. 上传 OCR 报告，检查候选映射并确认入档。
4. 注册第二个用户，演示亲友授权和代传。
5. 查看指标趋势图。
6. 提交机构评论。
7. 使用管理员账号审核评论并查看用户管理。

## 7. OCR 模式

完全离线演示时，在 `backend/.env` 设置：

```env
OCR_USE_MOCK=1
```

真实 OCR 模式需要网络和有效 OCR API 凭据，但不影响数据库的本地运行方式。

## 8. 常见问题

### 端口被占用

```powershell
.\scripts\start-full-dev.ps1 -BackendPort 5051
```

### 后端无法启动

确认虚拟环境存在：

```powershell
Test-Path .\backend\.venv\Scripts\python.exe
```

重新安装依赖：

```powershell
.\backend\.venv\Scripts\python.exe -m pip install -r .\backend\requirements.txt
```

### 数据库需要恢复

先关闭后端，然后用备份文件替换：

```text
backend/instance/health_system.db
```

### 数据库模型增加了新约束

先关闭后端，然后执行：

```powershell
Set-Location .\backend
.\.venv\Scripts\python.exe .\scripts\upgrade_local_database.py --check-only
.\.venv\Scripts\python.exe .\scripts\upgrade_local_database.py
```

升级前数据库会自动备份到 `backend/instance/`。

### 前端接口请求失败

检查后端健康接口：

```powershell
Invoke-RestMethod http://127.0.0.1:5050/api/health
```

预期结果：

```json
{"status":"ok"}
```
