# 康康健健 HealthDoc 本地运行与演示指南

本指南适用于当前第三轮重构后的健康服务平台：公开门户、三角色工作台、健康总览快捷测量、体检/自主记录时间线、同行体检安排、机构服务任务流、套餐版本与领域、亲友授权、报告识别导入和健康 AI，数据库版本为 SQLite schema v7。

## 1. 环境要求

- Windows 10/11、PowerShell 5.1+
- Python 3.10+
- Node.js 20+
- npm 10+
- 本地运行无需安装 MySQL、PostgreSQL 或其他数据库服务

## 2. 首次准备

在项目根目录执行：

```powershell
Set-Location .\backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

Set-Location ..\frontend
npm ci

Set-Location ..
if (-not (Test-Path .\backend\.env)) {
    Copy-Item .\backend\.env.example .\backend\.env
}
```

默认数据库：

```text
backend/instance/health_system.db
```

仓库跟踪这一份只含合成数据的 SQLite 演示快照，并同步跟踪 9 个带 `demo-v7` 水印的 PNG（3 个私有健康附件、6 个机构展示图），因此 fresh clone 后数据库引用的演示素材可直接使用。除这 9 个受控例外外，时间戳数据库备份、`backend/.env`、其余 `backend/uploads/` 运行时文件和 RAG 目录均不受 Git 跟踪。

## 3. 环境变量

离线演示建议保持：

```env
OCR_USE_MOCK=1
AI_USE_MOCK=1
```

真实 AI 需填写：

```env
AI_USE_MOCK=0
DEEPSEEK_API_KEY=你的密钥
```

真实华为云 OCR 需填写：

```env
OCR_USE_MOCK=0
HUAWEI_OCR_ENDPOINT=
HUAWEI_OCR_AK=
HUAWEI_OCR_SK=
HUAWEI_PROJECT_ID=
```

通知邮件默认使用 dry-run，不会连接外部邮箱：

```env
NOTIFICATION_EMAIL_DRY_RUN=1
```

需要真实投递时，配置 `SMTP_HOST/SMTP_PORT/SMTP_USERNAME/SMTP_PASSWORD/SMTP_FROM` 并把 dry-run 设为 `0`。注册必须填写有效邮箱，但邮箱只是通知渠道而不是账号唯一标识，多个家庭成员或演示账号可以共用同一个邮箱；空位提醒直接使用当前账号绑定的通知邮箱。邮件会以连续中文正文说明机构、日期、人数和下一步，不会发送 Outbox JSON 原文。本地演示可在被 Git 忽略的 `.env` 中设置 `DEMO_SHARED_EMAIL=你的测试邮箱` 统一绑定全部测试账号，也可用 `NOTIFICATION_EMAIL_REDIRECT` 临时把全部通知重定向到一个收件箱。真实邮箱不得写入受版本控制的文档或源码。

真实密钥只能写入 `backend/.env`。不要写入 `.env.example`、源码、截图、终端记录或文档。

## 4. 数据库检查与升级

启动前检查：

```powershell
Set-Location .\backend
if (Test-Path .\instance\health_system.db) {
    .\.venv\Scripts\python.exe .\scripts\upgrade_local_database.py --check-only
} else {
    Write-Host "数据库尚未创建，首次启动会创建 schema v7。"
}
Set-Location ..
```

预期当前输出包括：

```text
user_version=7
expected_user_version=7
schema_current=yes
```

如果版本不是 7，停止后端后执行：

```powershell
Set-Location .\backend
.\.venv\Scripts\python.exe .\scripts\upgrade_local_database.py
Set-Location ..
```

脚本会先生成带时间戳的结构升级备份，在临时库完成结构、行数、完整性和外键验证后替换正式库。

```text
backend/instance/health_system.before-schema-*.db
```

不要在确认不再需要前删除该备份。

## 5. 开发模式

在项目根目录执行：

```powershell
.\scripts\start-full-dev.ps1
```

脚本会打开后端 PowerShell 窗口，并在当前窗口启动前端：

- 前端：<http://127.0.0.1:5173>
- 后端：<http://127.0.0.1:5050>
- 健康检查：<http://127.0.0.1:5050/api/health>

后端健康检查通过后，脚本还会启动隐藏的通知 worker。它每 5 秒处理预约、约满和空位提醒 Outbox；退出当前前端命令时 worker 会一并停止。若只启动后端和前端两个独立脚本，邮件 worker 也需要单独启动：

```powershell
.\scripts\start-notification-worker.ps1
```

Vite 会把 `/api` 和 `/uploads` 代理到后端 `5050`。

## 6. 本机生产演示

```powershell
.\scripts\start-full-prod.ps1
```

- 前端：<http://127.0.0.1:4173>
- 后端：<http://127.0.0.1:5050>

脚本默认只监听回环地址。首次运行若缺少 JWT 密钥，会在被 Git 忽略的 `backend/.env` 生成随机值，并设置仅当前进程有效的本机演示标志。

不得把本机演示配置直接暴露到局域网或公网。对外监听至少需要：

- `JWT_SECRET_KEY` 长度不少于 32；
- `DEFAULT_ADMIN_PASSWORD` 长度不少于 12；
- 防火墙访问限制；
- HTTPS；
- 完成生产数据库 schema v7 迁移演练。

## 9. 推荐完整演示顺序

### 9.1 公开门户与认证

1. 打开 `/`，查看项目介绍、核心功能、使用流程、隐私提示和关于我们。
2. 打开登录、注册页，确认图片验证码正常加载。
3. 普通用户注册不需要邀请码；机构工作人员注册必须填写管理员签发的当前邀请码。
4. 用不同角色登录，确认分别进入用户、机构和管理员工作台。
5. 尝试跨角色 URL，确认进入 403 或被后端拒绝。
6. 在同一浏览器打开两个标签页，分别登录不同账号并刷新页面，确认两个标签页仍各自保持原账号；在一个标签页退出，另一个标签页不受影响。

### 9.2 普通用户完整流程

用 `test1` 登录：

1. 在“个人资料”确认健康身份码只读，可修改真实姓名、生日、性别、联系方式、过敏史和既往史。
2. 在“健康总览”查看健康摘要、领域卡片和最近动态，点击“记录测量”完成一次自主记录，不再进入独立测量板块。
3. 在“健康时间线”分别筛选“体检记录”和“自主记录”，确认本人录入的数据与机构归档结果都能看到；展开体检记录查看指标、文本结论和附件。
4. 在“健康趋势”按健康领域查看连续变化，确认同日存在机构结果时优先使用机构值，缺少机构指标时采用当天最后一次自主测量。
5. 在“体检预约”按机构、适用人群、健康领域和可约日期选择套餐；为本人或同行家人建立体检安排，满额时订阅空位提醒。
6. 返回“健康时间线”确认预约进度随取消、到检、报告归档等业务节点更新。
7. 浏览“体检机构”详情，确认机构介绍、服务能力、套餐版本、适用人群、包含领域、相册与评论都以服务产品方式呈现。
8. 在“亲友授权”查看或维护关系；授权后可切换亲友时间线/趋势，但不能代记测量或修改对方数据。
9. 在“我的评论”查看或删除本人评论；评论资格要求已有该机构发布的报告。
10. 打开健康 AI，先问普通问题，确认不加载报告；再主动引用报告并明确同意，观察 SSE 状态和回答。

### 9.3 机构账号完整流程

用 `institution1_staff1` 登录：

1. 查看机构运营总览及各预约状态数量。
2. 修改所属机构资料。
3. 维护套餐版本、适用人群和覆盖领域，提交新增、调整、下架或恢复申请；在“信息审核”查看处理进度或撤回。
4. 上传机构图片、拖拽排序并确认第一张成为普通用户封面。
5. 在“体检管理”从每日任务看板确认受检者到检，进入报告录入任务。
6. 录入标准指标、文本结论和附件，确认参考范围与异常提示；也可使用报告识别导入生成待复核内容。
7. 完成复核并归档；预约绑定用户停用时操作被拒绝，归档后内容不可编辑。
8. 确认预约同步变为已履约，普通用户的时间线、趋势和 AI 使用同一份归档结果。
9. 提交前确认页面明确提示不可撤回；归档后确认机构端不再显示撤下操作。
10. 使用同机构另一账号登录，确认它能管理同一机构数据；其他机构账号不能访问该报告。

### 9.4 报告识别导入流程

在体检管理页把预约确认到检后选择“识别导入”：

1. 保持 `OCR_USE_MOCK=1` 完成离线演示。
2. 提交 PDF/PNG/JPEG/WebP；受检者、日期和套餐自动取预约快照。
3. 查看识别候选、置信度、冲突、未匹配和需复核项目；技术诊断中保留 `parser_version` 便于排错。
4. 对需复核项手工修正或补充，不把未知字段直接当成标准指标。
5. 确认草稿至少有一项指标后锁定。
6. 锁定后确认报告不可修改，临时原文件已删除，用户侧无文件入口。
7. 提交后验证报告立即进入对应用户时间线，预约状态变为已履约。

### 9.5 系统管理员完整流程

用本机管理员账号登录：

1. 总览只显示账号、角色、机构、邀请码和评论等平台统计。
2. 新增、编辑、停用和恢复机构；维护相册，套餐只读查看。
3. 在“审核记录”处理机构套餐新增、修改、下架和恢复申请；确认通过后生效、驳回后不变。
4. 为机构生成邀请码，立即复制页面仅显示一次的明文。
5. 重新生成邀请码，确认旧码失效。
6. 查看账号，停用后确认该账号不能登录或刷新；再恢复。
7. 删除机构账号，确认历史报告仍保留创建用户名快照。
8. 删除普通用户时确认必须显式二次确认，并级联清理其健康数据。
9. 审核评论公开状态。
10. 确认管理员页面没有报告、自测、时间线、趋势、个人健康资料或健康 AI。

## 10. 预约履约与自动归档演示建议

1. 普通用户预约机构、套餐和日期，确认状态为未履约。
2. 机构确认到检，任务进入报告录入阶段；身份、日期和套餐版本由预约带入。
3. 录入指标、锁定并提交，确认报告进入用户报告列表、时间线和趋势，预约变为已履约。
4. 另建预约后先确认用户与授权亲友时间线显示未履约；由机构标记已失效，确认同一预约事件更新为“该预约已失效，请重新预约或联系机构”，且无法上传报告。

## 15. 常见问题

### 后端提示 schema upgrade required

停止后端，先执行 `--check-only`，再运行升级脚本。确认备份文件生成后重新启动。

### 前端请求失败或代理到错误端口

确认后端监听 `127.0.0.1:5050`，访问 `/api/health`。使用完整启动脚本时不要手工改代理；自定义端口需同步 `VITE_API_PROXY_TARGET`。

### 登录或刷新返回账号不可用

账号可能被管理员停用。使用管理员工作台恢复后重新登录，旧 token 不应绕过状态检查。

### 邀请码无法使用

确认机构启用、邀请码为当前 `active`、复制时没有多余空格，并且该码没有被使用或重新生成覆盖。

### 报告无法锁定，提示找不到用户

确认预约绑定账号仍为启用普通用户。身份、日期和套餐不允许在报告草稿修改；账号停用时锁定或提交返回 409。

### 报告无法编辑

只有 `draft` 可编辑。锁定后内容不可变；提交归档后不可撤下、删除或回到草稿。

### 普通用户看不到报告

确认报告是 `published`，`matched_user_id` 指向当前用户。用户只读接口不返回草稿或锁定报告。

### 趋势值与同日最后自测不同

同日存在已发布机构指标时，机构值优先；该指标未出现在报告中时使用当天最后自测。

### 报告识别返回错误或候选很少

确认文件格式、20 MiB 总大小、PDF 页数和 OCR 配置。未知或低置信度项目不会自动映射，需要人工复核或扩充指标字典。

### AI 提示未配置

### Waitress 提示 JWT_SECRET_KEY

推荐从根目录运行生产演示脚本，它会为回环演示生成密钥。直接调用 Waitress 时必须手工配置至少 32 字符的密钥。

### 可以直接运行服务器发布脚本吗

可以使用根目录 `scripts/deploy-server.ps1`；同步完整演示环境时显式增加 `-SyncDemoDatabase -SyncMailSettings`。脚本会同步合成数据库、9 个水印附件和 SMTP 白名单配置，并先备份 openGauss、上传目录、环境文件与当前版本，失败时整体回滚。
