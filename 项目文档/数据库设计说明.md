# 数据库设计说明

## 1. 运行方式

项目使用 SQLite，数据库文件为：

```text
backend/instance/health_system.db
```

应用启动时通过 SQLAlchemy 执行建表检查和种子数据初始化。测试使用独立的内存 SQLite。

所有 SQLite 连接强制启用外键，并设置 5 秒锁等待：

```sql
PRAGMA foreign_keys=ON;
PRAGMA busy_timeout=5000;
```

## 2. 表结构

| 表 | 用途 | 关键约束 |
|---|---|---|
| `users` | 用户和管理员 | 用户名唯一，邮箱唯一，角色检查 |
| `friend_relations` | 有方向的亲友关系与授权 | 用户对唯一，禁止自己添加自己 |
| `institutions` | 体检机构 | 机构名与分院名联合唯一 |
| `packages` | 机构套餐 | 机构与套餐名联合唯一，价格非负 |
| `indicator_categories` | 指标分类 | 分类名唯一 |
| `indicator_dicts` | 指标字典 | 指标编码唯一，分类与名称联合唯一 |
| `health_records` | 健康档案 | 归属人、上传人、状态检查 |
| `health_indicators` | 档案指标值 | 档案与指标联合唯一 |
| `comments` | 机构评论 | 评分范围 1–5 |

## 3. 主要关系

```text
users 1 ── N health_records（owner）
users 1 ── N health_records（uploader）
users 1 ── N comments
users 1 ── N friend_relations

institutions 1 ── N packages
institutions 1 ── N health_records
institutions 1 ── N comments

indicator_categories 1 ── N indicator_dicts
health_records 1 ── N health_indicators
indicator_dicts 1 ── N health_indicators
```

## 4. 关键字段

### `health_records`

- `owner_id`：档案归属用户。
- `uploader_id`：实际上传用户。
- `institution_id`、`package_id`：体检来源。
- `exam_date`：体检日期。
- `status`：`draft`、`parsed` 或 `confirmed`。
- `ocr_raw_text`：OCR 原始文本。

### `health_indicators`

- `record_id`：所属档案。
- `indicator_dict_id`：标准指标定义。
- `value`：原始指标值。
- `is_abnormal`：是否超出参考范围。
- `source`：`manual` 或 `ocr`。

### `friend_relations`

- `user_id`：发起添加关系的用户。
- `friend_user_id`：被添加用户。
- `auth_status`：被添加用户是否允许对方管理档案。

## 5. 完整性约束

数据库模型包含：

- 主键与外键。
- 非空约束。
- 单列和联合唯一约束。
- 角色、状态、评分、价格、值类型等检查约束。

当前物理结构包含：

- 22 个命名 CHECK 约束。
- `uq_institution_branch`。
- `uq_package_institution_name`。
- `uq_indicator_category_name`。
- 原有的用户、邮箱、指标编码、亲友关系和档案指标唯一约束。

跨表规则由后端事务校验，例如：

- 套餐必须属于档案选择的机构。
- 只有获得授权后才能代传亲友档案。
- 用户必须上传过某机构档案后才能评论。
- 指标数值根据字典参考范围计算异常状态。

## 6. 事务和并发

Flask-SQLAlchemy 为每个请求提供独立 Session。写操作成功后提交，失败时回滚。

SQLite 适合本项目的单机开发和课程演示。Waitress 可并发处理读取请求；写事务应保持短小，避免长时间占用数据库写锁。

## 7. 初始化

`backend/app/__init__.py` 在启动时执行：

1. 创建数据库表。
2. 写入默认管理员。
3. 写入体检机构和套餐。
4. 写入指标分类和指标字典。

种子逻辑可重复执行，不会重复插入已有数据。

## 8. 已有数据库结构升级

`db.create_all()` 只会创建缺失表，不会给旧 SQLite 表补加约束。因此模型结构变化后使用：

```powershell
Set-Location .\backend
.\.venv\Scripts\python.exe .\scripts\upgrade_local_database.py
```

脚本通过“新建临时库 → 按依赖顺序复制数据 → 完整性校验 → 备份原库 → 原子替换”的方式升级，避免直接修改旧表造成数据丢失。

当前 `health_system.db` 已完成升级：

- 缺失命名约束：0。
- 外键违规：0。
- `PRAGMA integrity_check`：`ok`。
- 升级前后所有业务表行数一致。

## 9. 备份与恢复

关闭后端后直接复制 `health_system.db` 即可备份。恢复时用备份文件替换当前文件。

报告附件不在数据库中，应同时备份：

```text
backend/uploads/
```
