### 2.1 认证与三角色隔离

覆盖：

- 图片验证码、普通注册、登录、访问令牌刷新和退出。
- user、institution_admin、admin 三种角色约束。
- 登录后按角色获取不同 dashboard summary。
- 普通用户、机构管理员、系统管理员接口互相隔离。
- 机构管理员只能绑定一个机构，一个机构最多一个管理员。
- 撤销机构管理员后立即失去机构接口权限，账号降级为普通用户且个人数据保留。
- 两类管理员不能把健康档案 ID 提交给健康 AI。

### 2.2 邀请码

覆盖：

- 系统管理员为指定启用机构签发邀请码。
- 邀请码明文只在签发时返回，查询不返回明文和哈希。
- 数据库仅保存哈希。
- 单次消费与机构管理员角色绑定。
- 已用、已撤销和被重新签发替换的旧码不可再次使用。
- 同一机构只有一个当前邀请码。
- 并发消费时只允许一个注册事务成功。
- 机构已有管理员或机构已停用时拒绝不合法签发/注册。
- 撤销管理员后重新任命必须使用新邀请码。

### 2.3 机构、套餐与相册

覆盖：

- 系统管理员新增、编辑、软停用和恢复机构。
- 机构管理员只能编辑所属机构。
- 系统管理员与机构管理员对套餐的新增、编辑、软停用。
- 停用机构时撤销机构管理员和当前未使用邀请码。
- 历史档案、套餐、评论和图片不因软停用丢失。
- JPEG、PNG、WebP 实际解码和重编码。
- 机构图片清除 EXIF，API 不暴露内部 storage_key。
- 单张 5 MB 上限在图片解码前拒绝超大请求。
- 每机构最多 8 张。
- 排序位置 0–7 唯一，第一张为封面。
- 图片删除与文件清理。
- 普通用户封面组件优先使用 `cover_image_url`，兼容旧 `logo_url`/图片集合，并在缺图或加载失败时提供可访问占位。

### 2.4 健康档案、OCR 与指标

覆盖：

- 手工档案和 OCR 档案可不选择机构、套餐。
- 选择套餐自动推导机构；清空机构同步清空套餐。
- 新档案不能关联已停用机构或套餐。
- 已有历史档案在来源机构停用后仍可进行不改变来源的合法维护。
- 数值清理空格、全角符号、方向箭头和单位文本。
- 不支持的单位不静默换算，进入人工确认。
- 指标字典映射、异常判断、唯一约束和趋势兼容。
- OCR Mock 解析、候选映射、确认入档。
- `region-v2` 按独立表格区域解析，多个表格复用第 0 行/列时不会发生单元格碰撞，且输入区域顺序不影响结果。
- 表格和非表格文本并行解析；元数据表不会抑制正文指标，无标准表头的代码/名称/结果表也可通过列特征推断。
- 纯英文短代码只接受精确代码/别名；验证类似长词或其他检验缩写不会因包含关系误映射到短代码指标。
- 同一指标多候选冲突、低置信度与非法值设置 `requires_review` 并退出自动确认；未配置指标保持未匹配，不猜成现有字典项目。
- 小数/千分位逗号按歧义拒绝，避免 `5,6` 静默变为 `56`；`μmol/L`、`µmol/L`、`umol/L` 和常见 `mol·L⁻¹` 字形按等价单位处理，未知单位不换算。
- 新建 OCR 档案和给既有档案补传报告的两种模式。
- 既有档案补传在确认前保留原状态、报告和指标；刷新后恢复当前候选，取消后恢复旧 OCR 快照。
- 既有档案 `replace_ocr` 会删除新报告未包含的旧 OCR 来源指标并保留手工来源指标；`merge` 只更新本次映射并保留其余旧 OCR 指标。
- 多页失败、成功但空结果、超过页数上限和旧版完整性缺失均禁止替换；新建不完整 OCR 档案必须显式二次确认。
- 随机 `attachment_id`、旧版本确认/取消、并发重传和授权撤销处理。
- 既有档案 OCR 补传/确认/取消和档案删除共享快照 CAS；冲突返回 409 并清理未采用、已替换或已删除的报告文件。直接新建档案覆盖普通 `parsed → confirmed` 流程。
- OCR 英文 `reference` 范围解析，避免把参考范围整体误写入指标值。
- 本人和授权亲友的档案 CRUD、代传权限。
- 档案响应 `display_id=health{id}`、指标响应 `record_display_id`，同时保持内部整数主键兼容。
- 原始报告只能从鉴权接口 /api/records/{id}/file 获取。
- 公共 /uploads 拒绝健康报告和未登记孤儿文件，仅允许数据库白名单中的机构图片。
- 报告与机构图片在数据库提交失败或删除后执行文件补偿清理。
- 普通用户和亲友代理不能转移既有档案归属。
- production 直接启动缺少强 JWT 密钥时拒绝；回环脚本可自动生成密钥并启用本机演示例外，对外监听仍拒绝演示管理员密码。

### 2.5 机构健康数据

覆盖：

- 机构管理员只能读取来源为所属机构且状态为 confirmed 的档案。
- 未关联机构、其他机构和未确认档案均不返回。
- 响应不含邮箱、手机号、上传人和原始报告。
- 机构健康接口仅支持 GET，不提供修改、补录或删除。
- 本机构档案详情、异常指标和趋势的范围限制。
- 健康数据访问生成结构化后端运行日志。

### 2.6 系统管理、评论与 AI

覆盖：

- 系统管理员全局档案监管接口。
- 用户列表显示角色和绑定机构。
- 评论提交门槛、用户维护、管理员审核、隐藏与删除。
- 匿名 AI FAQ、普通用户健康科普、急症安全分流和人工支持决策。
- `GET /api/ai/records` 仅返回本人/授权亲友、confirmed 且有指标的元数据，不泄露指标值。
- AI 档案每次重新鉴权；同一请求必须同一归属人，不设置数量上限，大批量 ID 按批加载。
- 单档全部指标；多档服务端确定性计算排序、首次/最新值、极值、异常、缺失、同日记录和数值变化。
- 非数值、缺失、同日多记录和不同日期无可比项不会强制生成趋势。
- SSE `meta/status/delta/action/done/error` 分帧、`select_records` 动作、请求 ID 和稳定错误结构。
- Waitress 响应不包含 `Connection` hop-by-hop 头；连接/读取/总超时、429/5xx、坏 JSON、空响应、首内容前重试、取消和客户端断开均有覆盖。
- 历史最多 20 条消息（10 个完整轮次）；满 20 条后确定性合并最早一轮，不再额外调用模型摘要，单次操作最多一次模型生成。
- prompt injection、急症规则和 decision-first JSON 安全边界。
- OCR 和 AI Mock 不消耗真实外部 API 额度。

| TC-M02 | 迁移失败 | 正式文件不被替换 |
| TC-M03 | v2 重复迁移 | 幂等通过 |

## 8. 外部服务说明

完整自动化回归使用 OCR Mock、AI Mock 和伪造 HTTP 响应验证接口契约，因此 CI 不依赖真实华为云或 DeepSeek 凭据。本地另使用不含真实健康信息的合成数据完成真实 DeepSeek 普通聊天、单档分析和多档分析 smoke test。

OCR 真实服务 smoke 使用三种布局而不是为某一张样图写死坐标：

| 合成报告布局 | 已配置指标映射 | 冲突 | 需复核 |
|---|---:|---:|---:|
| 中文有表头六列表格 | 10 | 0 | 0 |
| 中文双栏资料 + 无边框检验列表 | 9 | 0 | 0 |
| 英文行式结果 + reference 尾缀 | 8 | 0 | 0 |

三份结果均由真实华为云通用表格 OCR 返回后经过同一 `region-v2` 解析和映射链路。凭据、完整云响应和报告内容不进入仓库或测试输出。上述结果证明已配置指标在三种差异布局下可识别，但不承诺任意字典外项目自动映射；未知项目必须扩充字典或人工处理。
