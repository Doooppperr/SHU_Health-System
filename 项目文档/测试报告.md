## 2. 本轮重点验收

### 普通用户体验

- 日常测量不再作为独立导航板块，健康总览提供“记录测量”快捷入口。
- 旧 `/measurements` 地址兼容返回健康总览，不产生断链。
- 健康时间线统一展示“体检记录”和“自主记录”，支持 `record_type=all|exam|self` 筛选。
- 用户自主录入的数据可在时间线回看；体检记录可展开数值指标、文本结果和附件。
- 健康趋势按领域组织，同日机构结果优先，缺少机构值时回退当天最后一次自主测量。
- 预约按真实服务选择路径组织：机构、套餐版本、适用人群、健康领域、日期与参与者。
- 同行安排最多五人，满额时可订阅空位提醒；代预约不等于健康数据授权。

### 机构与平台体验

- 机构详情展示服务能力、套餐版本、适用人群、覆盖领域、价格、相册和评论，不再直接呈现数据库式套餐表。
- 机构工作台按到检、报告录入、结果复核、归档发布组织任务。
- 报告支持结构化指标、文本结论、附件及报告识别导入。
- 套餐新增、调整、下架和恢复进入审核流程，历史预约保留原套餐版本事实。
- 管理员只治理账号、机构、套餐审核、邀请和评论，不读取个人健康正文。

### UI 与响应式

- AI 入口位于工作台顶栏；桌面侧栏打开后内容按剩余空间重排，不整体缩放页面。
- 窄屏使用遮罩交互，关怀模式与工作台导航保持可读、可操作。
- 表格、卡片、筛选区和操作按钮覆盖常见桌面与窄屏宽度，避免 AI 入口、操作列和浮层重叠。
- 健康数据的日期范围控件被约束在所属弹性列内，不再覆盖相邻“资料来源”筛选项；样式源规则由回归测试固定。
- 空状态、加载状态、错误状态、403 与 404 均保留明确反馈。
- 服务器发布单元会同步校验合成演示附件、通知 worker 与 API 健康状态；任一环节失败即恢复发布前数据库、附件和环境配置。
