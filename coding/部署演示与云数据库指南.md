## 一、部署与答辩演示脚本

# 部署与答辩演示脚本

## 1. 云数据库前置步骤

当前演示数据库为华为云 GaussDB，前后端仍在本机运行。启动项目之前，先单独打开一个 PowerShell 窗口并保持 SSH 隧道：

```powershell
ssh -N -L 15432:192.168.0.31:8000 root@<ECS公网IP>
```

后端通过 `127.0.0.1:15432` 访问 GaussDB 内网地址 `192.168.0.31:8000`。如果这个窗口关闭，后端会无法连接云数据库。

`backend/.env` 应使用应用账号 `health_app`：

```env
DATABASE_URL=opengauss+psycopg2://health_app:<HEALTH_APP_PASSWORD>@127.0.0.1:15432/health_system?client_encoding=utf8
```

不要使用 GaussDB `root` 账号作为后端运行账号。

## 2. 一键启动脚本（项目根目录）

### 开发模式

```powershell
.\scripts\start-full-dev.ps1
```

- 后端：`http://127.0.0.1:5050`
- 前端：`http://127.0.0.1:5173`

### 生产演示模式


- 后端（Waitress）：`http://127.0.0.1:5050`
- 前端预览：`http://127.0.0.1:4173`

- 如需单独启动前端：`.\scripts\start-frontend-prod.ps1`

## 3. 演示账号准备

- 管理员（默认种子）：
  - 用户名：`admin`
  - 密码：`admin123`
- 普通用户：现场注册两个账号，例如：
  - `demo_user_a`
  - `demo_user_b`

## 4. 12分钟答辩演示流程（建议）

### 第1段：基础闭环（2分钟）

1. 用 `demo_user_a` 登录。
2. 进入机构列表 -> 机构详情，展示套餐数据。
3. 进入档案列表，新建1份个人档案并录入2-3个指标。

验收点：
- 已登录可访问业务页面。
- 机构/套餐和个人档案链路正常。

### 第2段：OCR闭环（2分钟）

1. 进入 `OCR上传` 页面，上传报告。
2. 展示“候选映射确认表”“未匹配字段”“过滤字段”。
3. 在候选映射表中可改选指标或忽略误匹配项。
4. 点击“确认入档”，跳转档案详情。

验收点：
- OCR上传解析成功。
- 状态可从 `parsed` 到 `confirmed`，且确认后指标才真正入档。

### 第3段：亲友授权闭环（3分钟）

1. `demo_user_a` 在亲友管理中添加 `demo_user_b`。
2. 切换 `demo_user_b` 登录，在“授权给我的关系”中打开授权。
3. 切回 `demo_user_a`，创建一份归属人为 `demo_user_b` 的档案（代传）。

验收点：
- 未授权时禁止代传，授权后允许代传。
- 档案归属人与上传人可区分显示。

### 第4段：趋势分析闭环（2分钟）

1. 进入“指标趋势”页面。
2. 选择归属人+指标，展示折线图与参考区间线。
3. 展示最小值/最大值/最新值统计。

验收点：
- 趋势按“归属人+指标”聚合。
- 支持本人和已授权亲友趋势查询。

### 第5段：评论审核闭环（3分钟）

1. `demo_user_a` 在机构详情提交评论（先确保有该机构上传记录）。
2. 用 `admin` 登录进入“评论审核”，将评论设为可见。
3. 回到机构详情，展示公开评论已出现。

验收点：
- 用户提交评论需满足“有该机构体检记录”门槛。
- 管理员可审核控制 `is_visible`。

## 5. 常见问题排查

1. 前端无法访问后端：确认后端监听 `5050`，并检查 Vite 代理配置。
2. 后端启动后数据库报错：确认 SSH 隧道窗口仍在运行，且 `backend/.env` 使用 `127.0.0.1:15432`。
3. 评论无法提交：先确认该用户上传过该机构档案。
4. 无法进入审核页：确认登录的是 `admin` 角色账号。
5. OCR映射少：样本可能不是指标页，可用手动录入补充演示。

## 6. OCR映射为0时的现场处理

当出现 `mapped_count=0`、`unmatched_count` 较高时，建议按以下话术和操作演示：

1. 先说明原因：
  - 该报告页可能包含机构信息、建议文本、参考范围碎片，非纯指标页。
2. 展示系统容错：
  - 页面会给出“未匹配字段”和“过滤字段”，不会误写入档案。
3. 使用候选映射确认表：
  - 若有候选项，手动改选后点击确认入档。
4. 无候选项时兜底：
  - 跳转档案详情页手动补录关键指标（FBG/TC/LDL/ALT/UA等），继续演示趋势与后续流程。

建议答辩话术：
- “系统在低质量或非指标页下优先保证正确性，先给出可审阅候选，再由用户确认入档，避免错误自动入库。”


## 二、华为云 GaussDB 迁移与运行指南

# 本地程序连接华为云 GaussDB

## 当前状态

项目当前采用“本地前后端 + 华为云 GaussDB 数据库”的方式运行：

```text
本机浏览器/前端 -> 本机 Flask 后端 -> SSH 隧道 -> 华为云 ECS -> GaussDB 内网地址
```

已完成事项：

- GaussDB 实例：`health_system`
- GaussDB 数据库：`health_system`
- 应用数据库用户：`health_app`
- GaussDB 内网地址：`192.168.0.31`
- GaussDB 端口：`8000`
- ECS 与 GaussDB 位于同一 VPC：`health_system`
- SQLite 历史数据已迁移到 GaussDB
- `password_encryption_type` 已调整为兼容普通 PostgreSQL/Python 驱动的模式，并已重置 `health_app` 密码

不要在文档中写入真实密码。真实密码只保存在本机 `backend/.env` 或由运行人员手动输入。

## 本地启动前置条件

因为 GaussDB 只有内网地址，本机必须通过 ECS 建立 SSH 隧道。单独打开一个 PowerShell 窗口：

```powershell
ssh -N -L 15432:192.168.0.31:8000 root@<ECS公网IP>
```

这个窗口需要保持打开。隧道开启后：

```text
127.0.0.1:15432 -> ECS -> 192.168.0.31:8000
```

## 云服务器/云数据库关闭后的重启流程

如果之前关闭过 ECS 或 GaussDB，重新使用时按顺序操作：

1. 在华为云控制台启动 GaussDB 实例，等待状态变为“运行中”。
2. 启动 ECS，等待云服务器状态变为“运行中”。
3. 确认 ECS 和 GaussDB 仍在同一个 VPC，GaussDB 内网地址仍为 `192.168.0.31`，端口仍为 `8000`。
4. 本机单独打开 PowerShell，重新建立 SSH 隧道：

```powershell
ssh -N -L 15432:192.168.0.31:8000 root@<ECS公网IP>
```

5. 另开一个 PowerShell 验证隧道：

```powershell
Test-NetConnection 127.0.0.1 -Port 15432
```

6. 看到 `TcpTestSucceeded : True` 后，再启动项目：

```powershell
Set-Location "H:\CODE\DA Project\health system"
.\scripts\start-full-prod.ps1
```

如果第 5 步失败，优先检查 SSH 隧道窗口是否还开着、ECS 安全组是否允许 SSH、GaussDB 和 ECS 是否仍处于同一 VPC。

## backend/.env 配置

本地 `backend/.env` 中数据库连接串应使用应用账号 `health_app`，不要使用管理员 `root`：

```env
DATABASE_URL=opengauss+psycopg2://health_app:<HEALTH_APP_PASSWORD>@127.0.0.1:15432/health_system?client_encoding=utf8
TARGET_DATABASE_URL=opengauss+psycopg2://health_app:<HEALTH_APP_PASSWORD>@127.0.0.1:15432/health_system?client_encoding=utf8
```

如果密码包含 `@`、`#`、`:`、`/` 等特殊字符，需要 URL 编码。例如 `@` 写成 `%40`。

## 迁移数据复现

迁移脚本位于：

```text
backend/scripts/migrate_sqlite_to_cloud.py
```

如果需要从本地 SQLite 重新迁移到 GaussDB，先确保 SSH 隧道已打开，然后在本机执行：

```powershell
Set-Location "H:\CODE\DA Project\health system\backend"
.\.venv\Scripts\Activate.ps1
python .\scripts\migrate_sqlite_to_cloud.py --source .\instance\health_system.db --replace
```

`--replace` 会清空目标库已有数据后再复制。执行前应确认云库中没有需要保留的新数据。

## 数据库级规则脚本

项目提供可重复执行的规则脚本，用于给现有 GaussDB 添加 `CHECK` 约束和触发器：

```powershell
Set-Location "H:\CODE\DA Project\health system\backend"
.\.venv\Scripts\python.exe .\scripts\apply_gaussdb_rules.py
```

脚本从 `backend/.env` 读取 `DATABASE_URL`，不会在仓库中保存真实密码。它会补充：

- 状态、评分、角色、价格、非空文本等 `CHECK` 约束。
- 档案套餐必须属于对应机构的触发器。
- 评论用户必须先上传该机构档案的触发器。
- 指标值写入时自动维护 `is_abnormal` 的触发器。

注意：`health_app` 是应用账号，只负责业务读写。给现有表添加约束和触发器需要管理员连接串。若使用 `root` 执行脚本时报 `none of the server's SASL authentication mechanisms are supported`，说明 `password_encryption_type` 调整后还没有重置 `root` 密码，需要先在 GaussDB 控制台重置 `root` 密码，再执行维护脚本。

当前云端 GaussDB 已执行该规则脚本并验证通过：

```text
CHECK 约束：22 个
触发器：3 个
应用连接账号：health_app
```

管理员账号仅用于本次 DDL 维护，不写入 `backend/.env`，也不作为后端运行账号。

## 账号边界

- `root`：GaussDB 管理员账号，仅用于建库、建用户、授权、维护。
- `health_app`：后端应用专用数据库账号，拥有项目表的读写权限。

后端程序必须使用 `health_app`，不要使用 `root` 作为运行账号。

## 依赖说明

项目依赖中已加入：

- `opengauss-sqlalchemy`：SQLAlchemy openGauss 方言
- `psycopg2-binary`：本地普通 PostgreSQL/Python 驱动
- `paramiko`：迁移脚本可选 SSH 隧道能力
- `PyMySQL`：仅保留给 GaussDB(for MySQL) 或 MySQL 兼容库使用

如果 GaussDB 参数未调整，普通 `psycopg2` 可能报：

```text
none of the server's SASL authentication mechanisms are supported
```

解决方式是将 GaussDB 参数 `password_encryption_type` 调整为兼容模式，并在参数生效后重置应用用户密码。

## 安全注意

- 不要把 ECS root 密码、GaussDB root 密码、`health_app` 密码写入 README 或提交到 Git。
- `backend/.env` 应保持在 `.gitignore` 中。
- 如果密码曾出现在聊天、截图或公开文档中，应及时在华为云控制台重置。
