## 10. OCR鲁棒入档改造专项验证（2026-04-10）

### 10.1 改造点

1. 上传阶段不再直接入库指标，改为输出候选映射并保存诊断信息。
2. 确认阶段支持提交人工确认映射列表（可改选、可忽略）后再入档。
3. 映射引擎新增噪声过滤与分层匹配（精确/去噪精确/包含匹配）。
4. 数值判异新增混合值提取（如 `5.6 mmol/L↑`）。

### 10.2 自动化回归结果

1. 后端OCR专项：
  - 命令：`cd backend && .venv\Scripts\python.exe -m pytest tests/test_ocr.py -q`
  - 结果：`4 passed`
2. 后端全量：
  - 命令：`cd backend && .venv\Scripts\python.exe -m pytest -q`
  - 结果：`25 passed`
3. 前端构建：
  - 命令：`cd frontend && npm run build`
  - 结果：`成功`

### 10.3 新增验证场景

1. 上传后状态为 `parsed` 且档案指标数为 `0`（未确认前不入档）。
2. 自动确认路径：未传 `confirmed_mappings` 时按阈值自动入档。
3. 人工确认路径：传入 `confirmed_mappings` 后仅按确认项入档。
4. 响应结构包含：`candidate_mappings`、`filtered_fields`、`diagnostics`。

### 10.4 结论

OCR 入档从“单步自动写库”升级为“候选映射可审阅后写库”，在多来源报告场景下显著提升了可控性与稳定性，且未引入对既有模块的回归问题。

## 11. GaussDB 迁移与规范化验证（2026-06-06）

### 11.1 云数据库迁移结果

当前运行数据库已从本地 SQLite 迁移到华为云 GaussDB。运行方式为：

```text
本地 Flask 后端 -> SSH 隧道 127.0.0.1:15432 -> ECS -> GaussDB 192.168.0.31:8000
```

应用数据库账号为 `health_app`，数据库名为 `health_system`。后端不使用管理员 `root` 账号连接数据库。

### 11.3 范式与约束

新增/确认候选键约束：

- `institutions(name, branch_name)`
- `packages(institution_id, name)`
- `indicator_dicts(category_id, name)`
- `health_indicators(record_id, indicator_dict_id)`
- `friend_relations(user_id, friend_user_id)`

数据库结构满足 3NF，核心字典表与关系表按 BCNF 候选键要求强化。详见 `coding/数据库规范化说明.md`。

```text

```

### 11.5 GaussDB 规则落库验证

当前云端 GaussDB 已补齐数据库级规则：

```text
CHECK 约束：22 个
触发器：3 个
```

已验证：

1. 评论评分越界会被 `ck_comments_rating_range` 拦截。
2. 档案 `package_id` 与 `institution_id` 不匹配会被 `trg_health_records_package_match` 拦截。
3. 写入异常指标值时，`trg_health_indicators_set_abnormal` 会自动维护 `is_abnormal`。
4. 验证过程在事务内回滚，未污染云端业务数据。
